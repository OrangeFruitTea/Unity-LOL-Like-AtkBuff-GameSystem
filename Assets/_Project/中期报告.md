## 1. 毕业设计的进展情况

### 1.1 课题工作完成情况

在项目中期，我们完成了以下核心工作：

**系统架构设计与实现：**
- 采用分层+模块化+MVC核心设计的架构方案，将系统划分为基础设施层、核心业务层、业务逻辑层、交互表现层四个层级
- 实现了基础设施层的引擎适配、资源管理、配置解析、日志调试等模块
- 实现了核心业务层的网络同步、数据存储、事件调度、时间管理等服务
- 实现了业务逻辑层的角色系统、对战规则、联机对战、信息管理等模块
- 实现了交互表现层的输入处理、用户界面、音效视效等模块

**核心系统开发：**
- **Buff系统**：实现了完整的Buff管理系统，包括Buff的添加、移除、更新、冲突处理等功能，采用对象池技术优化内存使用与性能
- **实体系统**：实现了基于ECS架构的实体管理系统，支持实体的创建、销毁、组件管理等功能
- **输入系统**：实现了键盘+鼠标的操作映射，支持自定义按键配置，确保操作的精准性与响应速度
- **UI系统**：实现了覆盖对局全流程的用户界面，包括大厅界面、战斗界面、角色信息界面等
- **网络同步系统**：基于Photon PUN 2实现了局域网联机功能，支持房间管理、玩家状态同步、对战指令传输等

**功能模块开发：**
- **角色系统**：实现了角色的属性管理、状态管理、技能释放等功能，每个角色具备独立的属性与技能，支持技能的等级提升与效果叠加
- **技能系统**：实现了技能的释放触发、弹道运动、命中判定、效果结算等完整逻辑，支持多种技能类型
- **对战规则系统**：实现了小兵刷新、补兵判定、经济收益、装备购买、资源争夺等核心对战规则
- **联机对战系统**：实现了房间创建、玩家加入、匹配开始、实时同步等功能，支持断连重连机制
- **信息管理系统**：实现了角色信息浏览、历史对局记录、数据统计与分析等功能

**性能优化与测试：**
- **资源优化**：采用资源池技术对高频实例化的资源进行缓存与复用，使用异步加载技术处理高耗时操作
- **网络优化**：采用状态同步+事件同步的混合策略，优化网络传输效率
- **代码优化**：优化代码结构，减少冗余代码，提高代码执行效率
- **测试验证**：进行了功能测试、性能测试、网络同步测试等，确保系统的稳定性与可靠性

## 2. 存在问题与解决方案

### 2.1 存在的主要问题

**技术层面问题：**
1. **多端网络同步一致性**：在局域网联机时，不同客户端之间的角色位置、技能状态、战斗数据同步存在延迟与误差，影响游戏体验
2. **资源管理与性能优化**：游戏运行过程中内存占用较高，在复杂场景或多人对战时帧率出现波动
3. **对战规则逻辑闭环**："小兵刷新-补兵判定-经济收益-装备购买-属性更新-资源争夺"的完整对战逻辑链实现复杂，易出现逻辑冲突或数据错乱
4. **AI系统开发**：AI角色的行为逻辑、路径规划、战斗策略等功能尚未实现，影响单机游玩体验
5. **回放系统开发**：历史对局的回放、分析、分享等功能尚未实现，影响用户对游戏的深度体验

**非技术层面问题：**
1. **项目进度管理**：部分功能模块的开发进度滞后，需要调整开发计划
2. **团队协作**：团队成员之间的沟通与协作需要进一步加强
3. **测试覆盖度**：部分功能模块的测试覆盖度不足，需要加强测试力度

### 2.2 解决方案与可行性研究

**技术层面解决方案：**
1. **多端网络同步一致性**：
   - 采用基于Photon PUN的"状态同步+事件同步"混合策略，核心数据（角色位置、技能释放指令）采用高频同步（10次/秒以上）
   - 对非核心数据（如小兵血量、经济数值）采用插值同步与事件触发同步结合，减少网络开销
   - 实现数据序列化压缩，降低网络传输数据量
   - 设计对局数据本地缓存机制，支持断连重连后快速恢复游戏状态
   - **可行性分析**：该方案已在同类局域网MOBA游戏中得到验证，技术成熟可靠，实施成本可控

2. **资源管理与性能优化**：
   - 采用"资源池+异步加载"的双层优化策略，对高频实例化对象使用对象池技术
   - 针对场景切换、技能特效加载等耗时操作，采用Unity协程实现异步加载
   - 使用Unity Profiler工具实时监控内存占用与帧率变化，针对性优化资源大小
   - 实现LOD技术，对远处的模型使用低精度版本，减少渲染开销
   - **可行性分析**：该方案属于Unity开发的常规性能优化手段，技术门槛较低，效果可量化

3. **对战规则逻辑闭环**：
   - 基于"事件驱动+模块化拆分"设计，通过核心业务层的事件调度服务实现各模块间解耦通信
   - 对战规则模块内部按"小兵机制、经济系统、资源争夺"拆分子模块，每个子模块封装独立逻辑
   - 将小兵刷新间隔、经济收益数值等参数配置化，便于调试与平衡优化
   - **可行性分析**：该方案符合软件工程模块化设计思想，本科阶段已通过课程设计积累相关经验

4. **AI系统开发**：
   - 采用行为树（Behavior Tree）或状态机（State Machine）实现AI角色的行为逻辑
   - 使用A*算法实现路径规划功能，支持寻路与避障
   - 设计简单的战斗策略系统，包括攻击、防御、逃跑等基本行为
   - **可行性分析**：AI系统是游戏开发中的常见功能，有成熟的实现方案与开源资源可供参考

5. **回放系统开发**：
   - 设计对局数据记录格式，记录玩家操作、角色状态、战斗事件等核心数据
   - 实现数据存储与回放功能，支持对局的回放与分析
   - 开发回放控制界面，支持播放、暂停、快进、慢放等操作
   - **可行性分析**：回放系统是游戏开发中的常见功能，技术实现难度适中

**非技术层面解决方案：**
1. **项目进度管理**：
   - 重新评估项目进度，调整开发计划，优先开发核心功能模块
   - 建立每周进度报告机制，及时跟踪项目进展情况
   - 采用敏捷开发方法，迭代式开发，快速响应用户需求
   - **可行性分析**：通过合理的项目管理方法，可以有效控制项目进度

2. **团队协作**：
   - 建立定期团队会议机制，加强团队成员之间的沟通与协作
   - 使用项目管理工具（如Trello、Jira）管理任务分配与进度跟踪
   - 建立代码审查机制，确保代码质量与一致性
   - **可行性分析**：通过有效的团队协作机制，可以提高开发效率与代码质量

3. **测试覆盖度**：
   - 制定详细的测试计划，包括功能测试、性能测试、兼容性测试等
   - 建立自动化测试流程，提高测试效率与覆盖度
   - 邀请外部测试人员进行游戏测试，收集反馈意见
   - **可行性分析**：通过系统的测试计划，可以有效提高测试覆盖度与游戏质量
