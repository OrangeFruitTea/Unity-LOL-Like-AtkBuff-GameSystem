# 局内单位死亡、复活延迟与出生点挂载指南

本文说明工程中 **ECS 击倒判定 → 死亡播报 → 软复活（含传送回出生点）** 的流水线，以及在 Prefab / 场景里应挂载哪些脚本、`Inspector` 如何填写。**不涉及玩法胜负**，仅覆盖「单位死亡后不销毁宿主、延时复活」的演示向配置。

---

## 1. 运行时流水线（理解后再挂载）

| 顺序 | 模块 | 作用 |
|------|------|------|
| 1 | `UnitVitalitySystem` | 每帧扫描 `EntityDataComponent.CrtHp`；降至阈值以下且存在 `CombatBoardLiteComponent` 时，补齐 `KillerEntityId`，并 **仅首次** 调用 `CombatUnitDeathRelay.AnnounceFirstDeath`。 |
| 2 | `CombatUnitDeathRelay` | `Debug.Log`、`UnitDeathEventHub.Raise`、`GameEventBus.Publish(CombatUnitDiedGameEvent)`。 |
| 3 | `UnitAnimDrv` | 订阅 `UnitDeathEventHub`：设置 Animator `IsDead`，本地 `_dead` 门闸阻止普攻动画链等。 |
| 4 | `DestroyHostOnUnitDeath` | 若 **未** 被软复活托管：按 `useDestroy` 选择 `Destroy` 或 `SetActive(false)`；若被托管则 **跳过**。 |
| 5 | `UnitDeathRespawnBehaviour` | 订阅同一 Hub：倒地后 **禁用**（可选）`NavMeshAgent`、`MovementController`，经 **`WaitForSeconds`（复活延迟）** 后执行回血、黑板复位、传送、`NotifyRevived()`，并派发复活事件（见 §5）。 |

**死亡倒计时**：当前实现为脚本内部的 **`yield return new WaitForSeconds(delay)`**，没有全局「剩余秒数」API。若要做屏幕倒计时，请见 §6。

## 3. Prefab 挂载清单

### 3.1 通用（所有要走「软复活」的单位）

在同一 **根物体**（带 `EntityBase`、与 ECS 绑定一致）上：

1. **`EntityBase`**（必选，已有）  
2. **`UnitDeathRespawnBehaviour`**（必选）  
   - **Kind**：英雄选 `Hero`，野怪 / 小兵选 `CreepOrMinion`。  
   - **Hero Respawn Delay Seconds / Creep Respawn Delay Seconds**：倒地至复活的等待秒数（即逻辑上的复活延迟）。  
   - **Refill Mp On Respawn**：是否在复活时把 `CrtMp` 拉回 `MpLimit`。  
   - **Suppress Host Destroy**：保持 **`true`**，以便同物体上的 `DestroyHostOnUnitDeath` **不会销毁/隐藏**宿主。  
   - **Disable Nav Agent While Dead / Disable Movement Controller While Dead**：建议英雄 **`true`**，避免尸体仍在导航或接收点击移动。  
   - **Nav Mesh Sample Radius**：复活传送点对 NavMesh 的投影搜索半径（米），与 `MovementController` 等保持同量级即可（默认 `4`）。  
   - 引用字段可为空：`Awake` 会尝试 `GetComponent`；若无 **`UnitSpawnAnchor`** 会自动 **`AddComponent<UnitSpawnAnchor>`**（仍建议在 Prefab 上显式挂一份便于改 `captureSpawnAtStart`）。  
3. **`UnitSpawnAnchor`**（强烈建议显式挂载）  
   - **Capture Spawn At Start = true**（默认）：在 **`Start`** 抓取当前世界坐标/朝向作为出生点。适用于：`JungleCampSpawner` 等 **`Instantiate` 后位置已正确** 的单位。  
   - **Capture Spawn At Start = false**：不在 `Start` 自动抓取；需在编辑器填 **`spawnWorldPosition` / `spawnWorldRotation`**，或运行时调用 **`SetSpawnWorldPose`**（例如泉水与初始站位不同）。  
4. **`UnitAnimDrv`**（若模型需死亡动画 / 复活后恢复 locomotion，已有则保留）  
5. **`DestroyHostOnUnitDeath`**（可选保留）  
   - 当 **`UnitDeathRespawnBehaviour.SuppressHostDestroy == true`** 时，本脚本收到死亡事件后会 **直接 return**，不产生销毁；无需删组件，便于以后改成「真死亡删物体」时只关软复活。  

### 3.2 仍要「死亡即删除」的单位（木桩、一次性物体）

- **不要**挂 `UnitDeathRespawnBehaviour`，或将其 **`suppressHostDestroy = false`** 且 **`enabled = false`**。  
- **`DestroyHostOnUnitDeath.useDestroy = true`**（默认）。

## 4. 出生点（泉水 / 营地）配置策略

| 场景 | 推荐配置 |
|------|-----------|
| 野怪：`JungleCampSpawner` 生成后不再挪动根节点 | `UnitSpawnAnchor.captureSpawnAtStart = true`，无需手写坐标。 |
| 英雄：需在 **泉水** 复活，而关卡里英雄初始 Transform 不在泉水 | `captureSpawnAtStart = false`，在 Inspector 填写出生世界坐标/朝向；或在开局 **`UnitSpawnAnchor.SetSpawnWorldPose(泉水.position, 泉水.rotation)`**。 |
| 生成后再微调位置 | 生成逻辑 **`Instantiate` + SetParent** 结束后，调用 **`CaptureFromCurrentTransform()`** 覆盖出生点。 |

复活时逻辑会使用 **`NavMesh.SamplePosition`** 将出生点投影到可走网格；采样失败则退回原始目标坐标（需注意离线高度）。

## 5. 复活瞬间会做哪些事（便于你对齐数值/UI）

对 **`BoundEcsEntity`** 与宿主 Transform：

- **`CrtHp`** ← **`HpLimit`**；可选 **`CrtMp`** ← **`MpLimit`**。  
- **`CombatBoardLiteComponent.InitializeDefaults()`**（清空击杀黑板槽位等）。  
- 若存在 **`JungleCreepModuleComponent`**：`CurrentState = Idle`。  
- 若存在 **`LaneMinionModuleComponent`**：**`WaypointIndex = 0`**（回到路径起点语义）。  
- **`NavMeshAgent`**：启用后 **`Warp`** + **`ResetPath`**（若存在 Agent）。  
- **`UnitAnimDrv.NotifyRevived()`**：清除本地死亡门闸、`IsDead = false`。  
- **`UnitRespawnEventHub.Raise(ecs)`**。  
- **`GameEventBus.Publish(new CombatUnitRespawnedGameEvent { UnitEntityId = ecs.Id })`**。

## 6. 「死亡倒计时」界面如何做（轻量化）

当前 **`UnitDeathRespawnBehaviour` 不向外推送剩余时间**。可选两种低成本做法：

**做法 A（推荐演示）：约定延时一致**

- UI 订阅 **`UnitDeathEventHub.UnitDied`** 或 **`CombatUnitDiedGameEvent`**，过滤 **`VictimEntityId == 本地英雄 ECS Id`**（或通过 `EntityEcsLinkRegistry` 比对宿主）。  
- 倒计时总时长与 Prefab 上 **`heroRespawnDelaySeconds`** **写成同一个常量**（或 ScriptableObject 配置），用 **`Coroutine` / `Update` 减秒** 显示；倒计时结束不必绑定复活帧（允许 1 帧误差）。

**做法 B：不写倒计时**

- 仅播放死亡 UI / 灰屏提示，复活出现时监听 **`UnitRespawnEventHub`** 或 **`CombatUnitRespawnedGameEvent`** 关闭遮罩。

若日后要把倒计时做成「绝对精确」，再在 `UnitDeathRespawnBehaviour` 增加 **`float RespawnAtUnscaledTime`** 或 **`UnitRespawnTickScheduledGameEvent`** 等扩展即可。

---

## 7. 与选中 / 属性面板的关系

死亡期间 ECS 实体仍存在，宿主物体未被销毁；若 UI 依赖 **`EntityBase` / `BoundEcsEntity.Id`**，一般 **无需解绑**。复活后 Id **不变**（软复活未 `Destroy` 宿主）。
