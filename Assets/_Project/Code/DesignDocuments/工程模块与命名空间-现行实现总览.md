# 工程模块与命名空间 — 现行实现总览

本文档依据仓库内 **`Assets/_Project/Code/Scripts`** 下 C# 脚本的**目录布局**与 **`namespace` 声明**，描述当前工程的功能模块划分与各模块职责。**不替代**专项设计（如技能表结构、Impact 形态、网络同步细节），仅作为「代码地图」与毕设/协作时的入口。

---

## 1. 范围与约定

| 项 | 说明 |
|----|------|
| 代码根路径 | `Assets/_Project/Code/Scripts` |
| 程序集 | 主工程脚本默认进入 **Assembly-CSharp**（本目录下未发现独立 `asmdef`）；第三方 **`ThirdParty/`**（如 Mirror）另有程序集边界，本文不写第三方细节 |
| 「模块」含义 | **逻辑域**：由一个或多个命名空间 + 顶层（无命名空间）类型共同承担的职责 |
| 命名空间≠文件夹 | **`Core.Entity` 命名空间大量使用于 `Gameplay/Entity/`** 等与 `Core/` 不同路径的文件中，读代码时需以 **`namespace` 为准** |

---

## 2. 顶层文件夹速览（物理）

```
Scripts/
├── Core/                    # ECS 内核、相机等少量横切
├── Basement/               # 基础设施：事件总线、JSON、配置、资源池、协程线程、运行时泵等
├── Gameplay/               # 局内战斗主体：实体、Impact、Targeting、Buff、技能、商店装备等（含 `Runtime/` 下 ECS 引导）
├── Presentation/           # 单位 Animator 驱动、受击/UI 钩子（与 ECS 并行，薄接线）
├── Network/               # Mirror 局域网/会话门面与事件 DTO（Gameplay.Network*）
└── Test/                   # Editor 测试 Runner 等（namespace Test）
```

## 4. 核心模块详解

### 4.1 `Core.ECS` — ECS 宿主与节拍

| 组件 | 功能 |
|------|------|
| **`EcsWorld`** (`Singleton<EcsWorld>`，`BeforeSceneLoad` 通过 `EcsWorldInitializer` 预热 `Instance`) | 持有 **`EcsEntityManager`**；`Initialize` **仅**做：`WarmStreamingGameTablesEarly`（拉起 **`JsonManager`/`LogManager`**、按需 **`BuffDataLoader`** GameObject，进程内首次 **`TryReloadSkillCatalogFromStreaming`** 填入 **`SkillCatalog`**；该静态方法与场景中 **`SkillDataLoader.Load()`** 共用）+ **`EcsManager` 构造**。各 **`IEcsSystem`** **不在此注册**，改由 **`Gameplay.Runtime.GameplaySystemsBootstrap`**（及 **`Basement.Runtime`** 下的 **`BasementRuntimeBootstrap`** 等）在 **`AfterSceneLoad`** **`AddEcsSystem`**；静态 `GetComponent` 等捷径；**`CombatImpacts`**/`ImpactManager`（懒创建） |
| **`EcsEntity` / `EcsEntityExtensions`** | Id、存在判断、`HasComponent`/`GetComponent`/`SetComponent` 等与 `EcsWorld` 绑定的扩展 |
| **`IEcsSystem`** | `Initialize`、`Update`、`Destroy`、`UpdateOrder` |

**现行 Gameplay 侧 `IEcsSystem` 注册**（**`GameplaySystemsBootstrap`**，`AfterSceneLoad`：逐项 **`TryAdd`**，仅在尚无该类型实例时 **`AddEcsSystem`**；运行时 **`EcsWorld.Update`** 中的先后由各 **`UpdateOrder`** 决定）：开局 **`BuffManager.EnableEcsDriving()`**（先于任意 **`BuffManager.Instance`**）；`EntitySpawnSystem` → `TowerCombatCycleSystem` → **`ImpactSystem`** → **`UnitVitalitySystem`** → **`BuffTickEcsSystem`** → `JungleAiSystem` → `LaneMinionMoveSystem` → **`SkillCastPipelineSystem`**（`UpdateOrder=50`）。同帧 **`BasementRuntimeBootstrap`** 可能追加 **`BasementPumpEcsSystem`**（`UpdateOrder=0`）；多个 **`AfterSceneLoad`** 回调之间的**源码执行顺序** Unity 不保证，ECS 帧内顺序以 **`UpdateOrder`** 为准。

---

### 4.2 `Core.Entity` + 子命名空间 — 局内实体与 MOBA-ish 兵种

单号映射 **Mono `EntityBase` ↔ ECS `EcsEntity`**：**`EntityEcsLinkRegistry`**、**`EntityEcsBridge`**（Buff/技能目标合法性等）。

**数据与黑板**

| 名称 | 作用 |
|------|------|
| **`EntityDataComponent`** | HP/攻击等 `EntityBaseDataCore` 键值 |
| **`CombatBoardLiteComponent`** | 主攻/仇恨/承伤/击杀者等逻辑 id 黑板 |
| **`FactionComponent`** / **`UnitArchetypeComponent`** | 阵营与小队 id、兵种 archetype |

**ECS 系统**

| 系统 | UpdateOrder（代码中声明） | 职责摘要 |
|------|---------------------------|----------|
| **`EntitySpawnSystem`** | 默认较早 | 消费生成请求、`IEntitySpawnExtension` 钩子 |
| **`TowerCombatCycleSystem`** | （见脚本） | 塔时间轴搜敌→写黑板 Strike→投 Impact |
| **`JungleAiSystem`** | （见脚本） | 野怪简易 AI |

**兵线 `Core.Entity.Minions`**：`LaneMinionModuleComponent`、`LaneMinionMoveSystem`、路点运行时等 stub/MVP。

**野区 `Core.Entity.Jungle`**：`JungleCreepModuleComponent`、`JungleAiSystem`、附着扩展。

**刷怪器 `Core.Entity.Spawn`**：`TowerSpawner`、`MinionWaveSpawner`、`JungleCampSpawner` — 面向场景的生成入口。

**生死：** **`UnitVitalitySystem`**：`CrtHp` 门槛、写 **`KillerEntityId`**；首次击倒经 **`Gameplay.Entity.CombatUnitDeathRelay`** 打 Log + **`UnitDeathEventHub`** + **`CombatUnitDiedGameEvent`** 上总线。

**可选宿主表现：** **`DestroyHostOnUnitDeath`** 监听 Hub 销毁根物体。

---

### 4.3 `Gameplay.Combat.Targeting` — 普攻与黑板对齐

选敌 **`DefaultTargetAcquisitionService`**、**`HostileTargetPicker`**、近战 **`MeleeStrikeRules`**、射程 **`CombatTargetingRange`**、`CombatBoardTargetSync`/`HostileAcquisitionCombatBoardAlign`。  
瞬时普攻投递：**`ICombatImpactDispatch`** 实现 **`DefaultCombatImpactDispatch`**（读 **`CombatBoardLite` Strike**，不经技能管线）。

---

### 4.4 `Core.Gameplay` + `Core.Combat` — Impact 伤害链

| 类型 | 职责 |
|------|------|
| **`ImpactManager`**（`Core.Gameplay`） | `CreateImpactEvent`、`AddModifier` 等；经 `EcsWorld.CombatImpacts` / `ImpactManager.Shared` 单例 |
| **`ImpactSystem`**（`Core.Combat`） | 每帧消费 Impact 事件实体，命中判定后改 `EntityData` 等 |

塔、普攻、`BuffOpcodeDispatcher`（`ImpactDamage`）等均经 `CombatImpacts` 投递，共用上述管线。

---

### 4.5 `Gameplay/BuffSystem/`（大部为**全局命名空间**）— Buff 宿主

脚本目录 **`Gameplay/BuffSystem`**，含：

| 代表类型 | 职责 |
|---------|------|
| **`BuffManager`** | 持有者维度 Buff 字典、`FixedDeltaTime` Tick、`AddBuff` |
| **`BuffPool`** / **`EffectManager`** | 池化与表现侧辅助（随项目迭代） |

---

### 4.6 `Gameplay.Skill*` — 表驱动技能与 Buff 接驳

门面 **`SkillExecutionFacade.TryBeginCast`** → 校验 + **`SkillCastPipelineSystem.TryBegin`**。  
管线：`PipelineScheduleBuilder` 分批 → `ExecuteStep` → `BuffApplyService.TryApplyStep` → `BuffTypeRegistry` 工厂（含 `MetaBuffApplyFactory`）→ `BuffManager`。  
`BuffOpcodeMvpBootstrap`：MVP opcode composition 注册。  
`SkillDataLoader` / `SkillCatalog`：JSON 技能定义载入（**表预热仍在 `EcsWorld.Initialize()`/`WarmStreamingGameTablesEarly`（`Awake`）完成，早于 `AfterSceneLoad` 的 ECS 系统注册**；场景中 Loader **`Load()`** 可二次覆盖）。

**缺口（代码占位）：** `OnEvent` 类步骤仍为 stub；位移/强制寻路/部分 opcode 在 **`BuffOpcodeDispatcher`** 中为占位日志。

---

### 4.7 `Gameplay.Shop` + `Gameplay.Equipment*` — 经济侧与装备Buff

商店购买流水、`ShopGameplayEvents`、锻造 `CraftingService` / `CraftRecipeCatalog`；英雄背上 `HeroEquipmentLoadout` 与 `EquipmentRuleEngine`。  
`EquipmentBuffApplier`：按 `EquipmentBuffBindingDefinition` 把装备关联到 Buff。

---

### 4.8 `Gameplay.Network*` — Mirror 会话封装

`GameplayNetworkManager`（Host/Join 门面）、`ILanLobbyBrowser`、`INetworkSession`、会话启停相关 `IGameEvent`（`Gameplay.Network.Events`）。

---

### 4.9 `Basement/*` — 横切基建

除 §3 表格外：**`Basement.MatchTime`** 对局时间与 Unity 驱动；**`Basement.Tasks`**/`Basement.Tasks.Unity` **定时任务**与 Unity 派发；**`Basement.Runtime`** **BasementUnityPump**、**`BasementRuntimeBootstrap`**（`AfterSceneLoad` 挂 **`BasementPumpEcsSystem`** 等）与 **`Gameplay.Runtime.GameplaySystemsBootstrap`** 同为「场景加载后」引导，职责分列 Basement / Gameplay。

## 5. 数据流鸟瞰（与本工程已实现部分）

```
技能 TryBeginCast
  → SkillCastPipelineSystem（定时步）
       → BuffApplyService → BuffManager → （表驱动 Buff，如 MetaBuff）
            → BuffOpcodeDispatcher（ImpactDamage opcode）
                 → CombatImpacts.CreateImpactEvent
                      → ImpactSystem → EntityData HP / CombatBoard.LastDamage …

普攻 / 塔 Strike
  → ImpactManager.CreateImpactEvent（不经技能管线）
       → ImpactSystem（同上）

HP ≤ 阈
  → UnitVitalitySystem → KillerEntityId
       → CombatUnitDeathRelay → Log + UnitDeathEventHub + GameEventBus (CombatUnitDiedGameEvent)
```

## 6. ECS 与非 ECS 的边界（便于检索）

| 机制 | 位置 |
|------|------|
| **ECS Tick** | `EcsWorld.Update` → 各 **`IEcsSystem`** |
| **Buff Tick** | **`Gameplay.BuffSystem.Ecs.BuffTickEcsSystem`**（`EcsWorld.Update`，`UpdateOrder≈33`）调用 **`BuffManager.Pump*`**；引导入口须先于实例化调用 **`BuffManager.EnableEcsDriving()`**（见 **`GameplaySystemsBootstrap`**）。未启用 ECS 驱动时，`BuffManager` 仍可用协程/`FixedUpdate` 自泵 |
| **技能管线节拍** | `SkillCastPipelineSystem.Update`（`Time.time` 相对施法起点） |
