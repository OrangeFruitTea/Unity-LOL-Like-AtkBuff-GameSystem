# 可视化资源与动画轻量化组织指南

| 项 | 内容 |
|----|------|
| 文档版本 | 2.0 |
| 适用引擎 | Unity 2022 |
| 适用场景 | **单人毕业作品 / 小体量 MOBA RPG Demo**：优先「少概念、拖拽为主、能快速录演示」 |
| 关联路径 | 与本项目 **`Assets/_Project/`** 结构兼容；Gameplay 中与 **`EntityBase` Prefab** 搭配即可 |

**一句话**：同一种东西永远放在**同一个父文件夹**，局内看见的**只靠 Prefab 串起来**；表里只记 **编号 id**，不写资源路径字符串。

---

## 1. 最简心法（先做这三件事）

单人开发不要一上来搭「资源平台」。按下面顺序做就足够支撑毕设演示：

```text
  [1] 定文件夹        →    [2] 做「局内Prefab」挂好模型/Animator/挂点    →    [3] 场景生成时只 Instantiate 这一条 Prefab
```

| 步骤 | 你要做什么 | 目的 |
|------|------------|------|
| **① 放哪** | 每个英雄/单位 **独占一个文件夹**（见 §3） | 找得到、不混资源 |
| **② 怎么挂** | **一个「局内用」根 Prefab** 里摆完：模型、Animator、空物体挂点（见 §4） | 场景里只拖这一份 |
| **③ 怎么配** | 程序里用 **整数 id**（如 `heroId=1`）选 Prefab；可选 **一张 ScriptableObject 表** 做 id→Prefab（见 §7） | 不写长路径、方便改表 |

动画、特效、图标都服从同一逻辑：**能进 Prefab 引用的就拖引用**；只有「太多张图」才用 **一张 SO 或 List 映射**（§5、§6）。

---

## 2. 目录结构（建议就这一版，不要再拆细）

在 `Assets/_Project/Art/` 下按用途分四块即可，毕设体量不需要更深：

```text
Art/
  Characters/              ← 角色：每个子文件夹对应一个 heroId（或起名 Hero_001）
    Hero_001/
      Source/              FBX（导入后尽量别改路径）
      Prefabs/             ★ 局内生成的「角色预制体」只认这里 ★
      Animations/          AnimatorController + 动画切片（少用可和 FBX 同放 Source）
      Materials/           材质（少做变体就不必再分子目录）
    Hero_002/
      ...
  VFX/                     ← 特效：全部是 Prefab，按 Skills / Common 分也行，随意但别散装
    Hit_A.prefab
  UI/
    Icons/                 Buff / Skill / HUD 图标，按大类分子文件夹即可
      BuffIcons/
```

**命名约定（写给自己看）**：文件夹名 **`Hero_001`** = 表里 **`heroId = 1`**（或与策划编号一致）；这样搜数字就能对上。

## 3. 角色：从 FBX 到「局内 Prefab」（最直观挂载）

### 3.1 你要得到的唯一入口

对战里生成的不是裸 Mesh，而是 **`Art/Characters/Hero_XXX/Prefabs/Hero_XXX_InGame.prefab`** 这一条。

### 3.2 推荐层级（记在纸上就能复现）

在 Unity 里打开 Prefab 编辑界面，层级尽量保持成这样（名字可自定，但要自洽）：

```text
Hero_XXX_InGame          ← 根：放移动、Entity、输入等「逻辑」脚本；Transform 作为整体位移
├─ Model                 ← SkinnedMeshRenderer、材质；子物体里挂 Animator 也行
├─ VFX_Hand / VFX_Foot   ← 空物体，只负责给特效当「挂点」
└─ (可选) Audio          ← 音效播放组件
```

**原则**：

- **根物体**管位移、旋转、逻辑；**不要**把战斗数值脚本塞进 `Model` 里面，避免以后换模要全改。
- **特效**永远 `Instantiate` 到 **`VFX_Hand` 的子级**（跟随手）或世界坐标点在脚下，二选一写在技能配置说明里即可。

### 3.3 动画放哪

| 简化做法（推荐毕设） | 说明 |
|---------------------|------|
| **Animator 挂在 `Model` 或根上** | 一个 **AnimatorController**（`.controller`）放在 **`Animations/`**，在 Inspector 里拖给 Animator |
| **参数少用** | 例如：`Speed`、`Attack`、`Dead` 三个就够跑 Demo；和程序约定 **常量类里写 Animator 参数名** |

不要做复杂分层（上肢 Mask 等）除非 Demo 明确要求；单层状态机更易排查。

---

## 4. 动画与程序怎么对齐（一页纸就够）

程序和美术各记这一张表即可（复制到记事本也行）：

| Animator 参数 | 类型 | 何时设 |
|---------------|------|--------|
| 例：`Speed` | float | 移动时 Mathf.Clamp |
| 例：`Dead` | bool/trigger | 死亡时 |

程序侧写一个 **`Animator` 字段**，在 **`Update` 或 `FixedUpdate`** 里把移动速度传给 `Speed`，比到处 `SendMessage` 清楚。

---

## 5. 图标：最省事两种做法（二选一）

**做法 A（最少脚本）**：UI 预制体上的 `Image.sprite` **直接在 Inspector 里拖**。适合图标很少、不写 Buff 大图鉴。

**做法 B（稍加一张表）**：建一个 **`ScriptableObject`**，里面 `List` 每项包含 **`int buffId`** + **`Sprite icon`**；UI 脚本里 `buffId` 查找列表。适合做技能说明界面。

**不推荐**：在项目里到处 `Resources.Load("路径")`——毕设存档换电脑路径一变就翻车。

---

## 6. 特效（VFX）

- **全部为 Prefab**：放在 `Art/VFX/`，起名如 `Slash_01.prefab`。
- **使用方式**：技能配置里只存 **`vfxPrefab` 引用**（若用 SO）或 **Prefab 名 + 小字典**；生成时在 **挂点 Transform** 下 `Instantiate`，或世界坐标生成。
- **毕设性能**：同屏特效不多时 **不必写对象池**；若卡顿，再给常用 2～3 个特效加简单 **复用**（禁用时 `SetActive(false)` 别 Destroy）。

---

## 7. 配置：id 怎么变成 Prefab（推荐唯一扩展点）

单人项目**只要一个**「总表」即可，例如 **`HeroVisualLibrary.asset`**（ScriptableObject）：

| 字段示例 | 含义 |
|----------|------|
| `heroId` | 与 JSON/关卡配置相同 |
| `characterPrefab` | 拖 `Prefabs/Hero_XXX_InGame` |
| `portrait`（可选） | 头像 Sprite |

**生成角色时**：`heroId → 表里查到 Prefab → Instantiate`。  
不想写 SO 也行：**场景中放一个单例**，上面 **public 数组**，在 Inspector 里按顺序摆 Prefab，`heroId` 当数组下标——毕设可以接受，记得注释写清楚。

---

## 8. 从建资源到进场：一条流水线（可按表操作）

按顺序勾选即可：

1. FBX 进 `Characters/Hero_XXX/Source`，Rig 设 Humanoid（人形）。
2. 调好材质，拖到场景里摆出 **Prefab** → 存到 **`Prefabs/Hero_XXX_InGame.prefab`**。
3. Animator 指定 **`.controller`**，连好 Idle/Walk（或单层 AnyState）。
4. 在 **`HeroVisualLibrary`**（或生成器数组）里 **填一行**：id + Prefab 引用。
5. 关卡里：读 id → Instantiate → `GetComponent` 你的移动/战斗脚本。
6. 技能 VFX：**技能配置**里拖入 VFX Prefab；或由技能 id 在小表里查 Prefab。
7. UI 图标：按 §5.A 或 §5.B 接上。

---

## 9. 常见坑（单人毕设避雷）

| 现象 | 建议 |
|------|------|
| 换了个模型整条逻辑崩 | 脚本挂在 **稳定的根**，不要绑在可能被删的 Mes 子物体上 |
| 动画抽搐 | Avatar 不匹配或 Animator 里缺 State；先有 Idle 再往复杂做 |
| 找不到 Prefab | 坚持 **Prefab 永远在 `Characters/XXX/Prefabs`**，别 Copy 到别处双份 |
| 配置太散 | 尽量 **id 只出现两处**：数据表 + `HeroVisualLibrary` |

---

## 10. 什么时候再考虑复杂方案

仅当毕设之外还要继续扩展时，再考虑：**Addressables**、**图集自动打包**、**特效对象池框架**。当前文档以 **「能做完、能演示、好改」** 为第一目标。

## 12. 毕设演示前自检（可打印）

- [ ] 每个英雄 **只有一个局内入口 Prefab**，路径固定  
- [ ] `heroId` 与文件夹名 / 总表 **一致**  
- [ ] Animator 参数名在 **代码常量** 里有一份  
- [ ] 技能、Buff 要么 Inspector 拖图，要么 **一张表** 映射，没有第三种到处写路径  
- [ ] 录屏前 **Build 一次** 确认没有 Missing Reference（红色 None）
