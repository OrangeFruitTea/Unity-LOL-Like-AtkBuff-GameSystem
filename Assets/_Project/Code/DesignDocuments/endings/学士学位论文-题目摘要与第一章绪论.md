## 摘要（中文）

多人在线战术竞技类游戏在玩法上牵涉角色、战场与规则等多方面要素，若以软件制品交付，往往需要清晰的架构边界与稳固的运行底座。本文为本科毕业设计论文，课题为基于 Unity 引擎的俯视角 MOBA 游戏的设计与实现。全文在梳理对战核心、交互辅助与信息服务等功能需求与非功能需求的基础上，给出分层模块化、事件驱动的总体结构。

论文的撰写与实现的**着力点首先放在已在工程中落地的底层与核心服务模块**，具体包括：对 Unity 引擎能力的适配封装、资源生命周期与加载策略管理、配置文件解析与统一管理、开发与运行期的日志诊断、全局事件分发、时序任务调度以及本地化数据读写等组成部分；上述模块在项目代码中具有明确的接口边界与脚本实现，可作为后续局内玩法与上层界面联调的稳定支撑。在此基础上，正文在**第二章**交代 Unity 工作流鸟瞰后，自**第三章**起依次陈述需求论证与总体方案、工程基础层要点、局内系统与交互表现设计，最终在**第七章**以系统验证与全文归纳收口。本文工作展示了「先夯实基底、再行扩展」类游戏客户端的开发路径，对相关实践具有一定参考意义。

## ABSTRACT（英文）

Top-down multiplayer online battle arena games involve multiple intertwined gameplay concerns; delivering them as dependable software favors clear layering and solid runtime foundations. This undergraduate thesis tackles the design and implementation of such a Unity-based prototype.

Functional and non-functional requirements are organized around combat-centric features, auxiliary interaction, and information services, yielding an overall layered modular architecture guided by explicit events. The exposition **emphasizes the infrastructure and core service modules that already exist as concrete implementations in the codebase**: adapters over Unity APIs, resource lifecycle and loading strategies, configuration parsing and management, diagnostics logging, dispatch of global events, time-based task scheduling, and localized data access. Together they provide contracted services for higher-level gameplay logic and presentation. After the introduction, Chapter 2 gives a concise **primer on Unity workflows** for unfamiliar readers; Chapters 3–6 present requirements planning, layered implementation oriented to deployed infrastructure, in-game logic and presentation bound to those contracts; Chapter 7 closes with verification and a recap. Adopting a “foundation first” client development path is illustrative for similar student projects coupling engine technology with disciplined software engineering structure.

# 第一章 绪论

## 1.1 研究工作的背景与意义

多人在线战术竞技品类在商业与教学场景中均常被采用，其核心体验建立在对抗、博弈与瞬时决策之上。从软件角度看，若以可维护、可追溯的方式交付同类产品，必须把**运行时底座**放在首位：若没有与引擎适度解耦的适配层、没有对资源与配置的统一管控、没有对事件与时序的稳定调度，上层任何玩法扩展都易被绑死在引擎细节上，也难以在单机调试与团队协作中保持稳定节奏。

正因如此，选取「俯视角 MOBA」一类课题作为本科毕业设计的载体，其价值并不止于题材本身，而更在于：**在有限周期内需完成一批可编译、可调测、可被其他模块调用**的底层与中台式能力，从而倒逼需求分析、模块划分与接口契约的规范化。电子科技大学软件工程专业强调理论与实践并进；将一个游戏形态的客户端拆解为可从代码仓直接指认的基础设施与核心服务，再渐次叙述局内与用户界面等环节，能够使课程中的面向对象、常见设计模式与质量属性用语落到可检查的制品上。

综上，本文以 MOBA 为应用语境，而将论述重心对齐到「**大型交互式软件的底座建设**」与「个人能力在真实规模脚本工程中的综合运用」，这既符合业界对客户端工程起步阶段的普遍做法，也契合本科毕业设计对工作量与可追溯成果的要求。

## 1.2 与本课题相关的研究与技术综述

商用与教学向的游戏客户端普遍采用**关注点分离**与**适配层隔离引擎 API**：将美术资源、运行时配置与逻辑判定分别纳入可校验的流水线，让**改配置、改表不必动核心代码**，也方便**做测试和问题定位**。观察者、发布订阅等模式常用于削弱模块编译期耦合，这一思路与信息系统中的中介与事件驱动实践相通。

Unity 已形成较为完备的编辑器与工作流文档，预制体体系、生命周期回调与 Profiler 等工具构成了本科阶段可用的主要工程支撑。为适应多实体场景中逻辑组织的清晰性，ECS 范式或其与面向对象脚本混用的做法在行业内广泛讨论；本文对相关概念仅作方法论层面的参照，不把某一范式的穷尽实现作为绪论评述的核心。

总体而言，最接近本课题的资料来自引擎手册、开源或教学向的模块化实践总结，以及通用软件工程教材中对内聚耦合、演化与调试友好性的论述。绪论在技术路线讨论上**聚焦于单机客户端结构、底层服务化边界与质量体系**，从而为后文对已落地模块的逐项说明提供概念先导。
