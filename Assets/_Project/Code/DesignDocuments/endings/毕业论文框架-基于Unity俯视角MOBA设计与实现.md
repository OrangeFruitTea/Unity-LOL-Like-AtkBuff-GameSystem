# 毕业论文框架：基于 Unity 的俯视角 MOBA 游戏设计与实现

本文档对齐《电子科技大学学士学位论文撰写范例》：**绪论** → **相关知识简介（独立装订、不占现行第二章序号）** → **分章正文六章**，各章含「本章小结」；第六章末可兼全文归纳并与院系装订的「结论」页对接。结构与写作重心见下文「撰写侧重点」；正文技术叙述以 **`DesignDocuments/` 姊妹文档与本工程自有脚本** 为准。

**路径说明**：本文件位于 `DesignDocuments/endings/`；文中 `../` 表示上级 `DesignDocuments/` 目录下的姊妹设计文档。

## 〇、撰写侧重点（动笔前可看）

建议将笔墨集中在 **需求与总体架构**、**基础设施与核心横切服务的工程化描述**、**局内子系统的分层设计说明**、**交互与界面设计及图示**。绪论可走教材与文献口吻；**第三章专写本仓库 Basement / 自持核心服务**，不单独展开第三方联机库。**表现层章节**可多配流程图、界面草图或编辑器截图。技术论据主要来自上级目录 **`DesignDocuments/`** 及 `_Project/Code/Scripts`。

| 优先级 | 建议展开方式 |
|--------|----------------|
| 需求与分层架构 | 按「对战核心—交互辅助—信息服务」模块化叙述，辅以架构图与模块职责表 |
| 基础设施（第三章） | 事件总线、配置与 JSON、资源池、`TimingTaskScheduler`、运行时引导、本地化数据链路；参阅各 `*Module.md` / `Basement`、`DataStorageService` 等相关设计文档 |
| 局内系统设计 | [`../工程模块与命名空间-现行实现总览.md`](../工程模块与命名空间-现行实现总览.md)：ECS、Impact、Targeting、Buff/技能管线等，用**系统设计、数据流与控制流**句式叙述 |
| UI 与视听 | 操作流程、控件布局原则、图示与案例分析 |
