## 4.4.1 资源池与对象复用

对局中角色生成、弹道、特效等会产生大量 `Instantiate` / `Destroy`（或等效分配）。若依赖运行时频繁申请与释放堆内存，易造成分配尖峰并触发集中 GC，拉高帧耗方差。对象池以**缓存空闲实例**换取更平滑的分配曲线（空间换时间），配合生命周期钩子清理残留状态，保证复用语义稳定。

工程上通过 `ResourcePoolManager`、`GameObjectPool`、`GenericResourcePool` 与 `ResourceLoader` 等模块封装预制体及关联资源的**获取—使用—回收**流程。

### 复用契约（`IReusable`）

```
接口 IReusable:
    过程 OnSpawn()    // 自池中「取出、即将投入使用」时
    过程 OnDespawn()  // 「归还入池」前，恢复可复用状态
```

## 4.4.2 底层队列式池（`SafeObjectPool<T>`）

```
类型 SafeObjectPool<T>，约束 T 为引用类型:
    字段 _idleObjects: 队列<T>      // 空闲实例
    字段 _createFunc: Func<T>      // 无空闲时新建
    字段 _resetAction: Action<T>   // 归还前重置（可为空）
    字段 _maxCapacity: int         // 空闲侧上限

过程 get() -> T:
    加锁
    若 _idleObjects 非空:
        返回 Dequeue()
    解锁
    返回 _createFunc()            // 超出空闲上限仍可通过新建满足借用需求

过程 Release(obj):
    若 obj 为空则抛错/返回（工程内为抛错）
    调用 _resetAction(obj)
    加锁
    若 _idleObjects.Count < _maxCapacity:
        Enqueue(obj)
    // 否则丢弃引用，由 GC 回收（控内存峰值）

过程 TryGet(out obj) -> bool:
    加锁
    若 _idleObjects 非空:
        obj = Dequeue(); 返回 true
    obj = null; 返回 false

过程 PreloadInitial(initialCapacity):
    循环 initialCapacity 次: Enqueue(_createFunc())

过程 Clear():
    加锁清空 _idleObjects          // 不销毁已外借对象，由业务层负责
```

## 4.4.3 资源加载（`ResourceLoader` / `IResourceLoader`）

```
类型 ResourceLoader 实现 IResourceLoader

过程 LoadSync<T>(resourcePath) -> T，约束 T : UnityEngine.Object:
    #if UNITY_EDITOR
        返回 Resources.Load<T>(resourcePath)
    #else
        handle = Addressables.LoadAssetAsync<T>(resourcePath)
        handle.WaitForCompletion()
        返回 handle.Result

过程 LoadAsync<T>(resourcePath, onLoaded, onProgress?):
    #if UNITY_EDITOR
        asyncOp = Resources.LoadAsync<T>(resourcePath)
        asyncOp.completed += (op) => onLoaded(request.asset as T)
    #else
        handle = Addressables.LoadAssetAsync<T>(resourcePath)
        可选: handle.Completed += onProgress
        handle.Completed += (op) =>
            若成功则 onLoaded(op.Result) 否则 onLoaded(null) 并打错误日志

过程 Unload<T>(resource):
    #if UNITY_EDITOR
        Resources.UnloadAsset(resource)
    #else
        Addressables.Release(resource)

过程 UnloadUnused():
    Resources.UnloadUnusedAssets()
```

## 4.4.4 游戏对象池（`GameObjectPool`）

```
类型 GameObjectPool 实现 IResourcePool<GameObject>
    字段 _internalPool: SafeObjectPool<GameObject>
    字段 _poolParent: Transform   // 默认新建名为 "GameObjectPool" 的隐藏父节点
    字段 _usedCount: int

构造(GameObject prefab, poolParent?, initialCapacity, maxCapacity):
    _poolParent = poolParent 或 new GameObject("GameObjectPool").transform
    _poolParent.gameObject.SetActive(false)

    createFunc = () =>
        obj = Instantiate(prefab, _poolParent)
        obj.name = prefab.name
        obj.SetActive(false)
        返回 obj

    resetAction = (obj) =>
        obj.transform.SetParent(_poolParent)
        obj.SetActive(false)
        重置 localPosition / localRotation / localScale 为默认
        若 obj 上存在 IReusable: 调用 OnDespawn()

    _internalPool = new SafeObjectPool(createFunc, resetAction, initialCapacity, maxCapacity)

过程 Spawn(position, rotation, parent?) -> GameObject:
    加锁
    obj = _internalPool.get()
    obj.transform.SetParent(parent, worldPositionStays: false)
    obj.position = position; obj.rotation = rotation
    obj.SetActive(true)
    _usedCount++
    若存在 IReusable: OnSpawn()
    返回 obj

过程 TrySpawn(out obj, ...):  // 与 Spawn 类似，但内层 TryGet，失败则 out=null

过程 Despawn(obj):
    若 obj 为空则返回
    加锁
    _internalPool.Release(obj)     // 内部先 OnDespawn（见 resetAction）
    _usedCount = max(0, _usedCount - 1)

过程 Preload(count):
    加锁
    重复 count 次:
        obj = _internalPool.get()
        _internalPool.Release(obj) // 借出后立即归还，扩充空闲队列

过程 Clear():
    加锁
    _internalPool.Clear()
    _usedCount = 0
```

## 4.4.5 泛型资源池（`GenericResourcePool<T>`）

```
类型 GenericResourcePool<T> 实现 IResourcePool<T>，约束 T : class
    字段 _internalPool: SafeObjectPool<T>
    字段 _onSpawnAction, _onDespawnAction: Action<T>（可选）

构造(createFunc, onSpawnAction?, onDespawnAction?, initialCapacity, maxCapacity):

    wrappedCreate = () =>
        obj = createFunc()
        若 obj 是 IReusable: reusable.OnSpawn()   // 仅「新建实例」路径
        返回 obj

    wrappedReset = (obj) =>
        若 obj 是 IReusable: reusable.OnDespawn()
        调用 onDespawnAction?(obj)

    _internalPool = new SafeObjectPool(wrappedCreate, wrappedReset, initialCapacity, maxCapacity)
    保存 _onSpawnAction = onSpawnAction

过程 Spawn() -> T:
    加锁
    obj = _internalPool.get()
    _usedCount++
    调用 _onSpawnAction?(obj)   // 每次取出后可选回调（可与 IReusable 组合使用）
    返回 obj

过程 TrySpawn(out obj): 类似，TryGet 成功后 _usedCount++ 并 _onSpawnAction

过程 Despawn(obj): 同 GameObjectPool，Release + 递减 _usedCount

过程 Preload / Clear: 与 GameObjectPool 模式相同（循环 get/Release；Clear 清空空闲队列）
```

**与 `GameObjectPool` 的差异（写作时可一笔带过）：** `GameObjectPool` 在每次 `Spawn` 后**统一**对挂载的 `IReusable` 调用 `OnSpawn`；`GenericResourcePool` 中 **`IReusable.OnSpawn` 在内层工厂包装里仅对「首次 `createFunc` 创建」调用**，从空闲队列复用时依赖可选的 `_onSpawnAction` 做额外初始化——工程中若类型实现 `IReusable`，通常应在创建池时传入与 `OnSpawn` 语义一致的 `_onSpawnAction`，以保持与「取出即初始化」叙述一致。

## 4.4.6 资源池管理器（`ResourcePoolManager`）

```
单例 ResourcePoolManager : Singleton<ResourcePoolManager>
    字段 _gameObjectPools: 字典<string, IResourcePool<GameObject>>
    字段 _genericPools: 字典<string, object>   // 装箱存各类 GenericResourcePool<T>
    字段 _resourceLoader: IResourceLoader

过程 Initialize(loader?):
    _resourceLoader = loader 或 new ResourceLoader()

过程 Awake():
    base.Awake(); Initialize()   // 默认自行 new ResourceLoader

// —— GameObject 池 ——

过程 CreateGameObjectPool(poolKey, prefab, initialCapacity, maxCapacity) -> GameObjectPool:
    加锁
    若 poolKey 已存在: 打警告并返回已有池
    pool = new GameObjectPool(prefab, null, initialCapacity, maxCapacity)
    _gameObjectPools[poolKey] = pool
    返回 pool

过程 CreateGameObjectPoolFromResource(poolKey, resourcePath, ...):
    prefab = _resourceLoader.LoadSync<GameObject>(resourcePath)
    若 prefab 为空: 报错并返回 null
    返回 CreateGameObjectPool(poolKey, prefab, ...)

过程 SpawnGameObject(poolKey) -> GameObject:
    // 重载: 可带 position, rotation, parent
    加锁
    若 _gameObjectPools[poolKey] 为 GameObjectPool: 返回其 Spawn(...)
    否则报错并返回 null

过程 DespawnGameObject(poolKey, obj):
    加锁
    若存在池: pool.Despawn(obj); 否则报错

// —— 泛型池 ——
// 键规则：fullKey = typeof(T).Name + "_" + poolKey

过程 CreateGenericPool<T>(poolKey, createFunc, onSpawn?, onDespawn?, initialCapacity, maxCapacity):
    加锁
    fullKey = typeof(T).Name + "_" + poolKey
    若已存在: 警告并返回已有
    pool = new GenericResourcePool<T>(createFunc, onSpawn, onDespawn, initialCapacity, maxCapacity)
    _genericPools[fullKey] = pool
    返回 pool

过程 SpawnGeneric<T>(poolKey) -> T:
    fullKey = ...; 查找 GenericResourcePool<T> 并 Spawn()

过程 DespawnGeneric<T>(poolKey, obj):
    fullKey = ...; 查找并 Despawn(obj)

// —— 预加载与清空 ——

过程 PreloadGameObjectPool(poolKey, count):
    若存在: pool.Preload(count)

过程 PreloadGenericPool<T>(poolKey, count):
    fullKey = ...; 若存在: genericPool.Preload(count)

过程 ClearAllPools():
    加锁
    对每个 GameObject 池: pool.Clear(); 清空 _gameObjectPools
    对每个泛型池: 若实现 IDisposable 则 Dispose(); 清空 _genericPools
        // 工程内 GenericResourcePool 当前未实现 IDisposable，此行保留与源码 `is IDisposable` 判断一致

过程 GetResourceLoader() -> IResourceLoader:
    返回 _resourceLoader
```

## 4.4.7 业务侧使用范式（获取—使用—回收）

```
// 初始化阶段：注册池（可直接握 prefab，或经 ResourceLoader 同步加载）
ResourcePoolManager.Instance.Initialize(可选自定义 IResourceLoader)
pool = CreateGameObjectPool("Vfx_Hit", hitPrefab, initialCapacity: 10, maxCapacity: 100)
// 或
pool = CreateGameObjectPoolFromResource("Vfx_Hit", "Prefabs/FX/Hit")

可选: PreloadGameObjectPool("Vfx_Hit", 20)   // 开局暖机，摊薄首帧实例化

// 运行时：每帧 / 事件中
go = SpawnGameObject("Vfx_Hit", pos, rot, parent)
... 使用完毕 ...
DespawnGameObject("Vfx_Hit", go)
```
