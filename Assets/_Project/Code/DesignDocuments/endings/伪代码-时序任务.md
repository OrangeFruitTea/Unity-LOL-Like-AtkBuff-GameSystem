## 4.5.2.1 设计意图与职责边界

玩法中技能冷却、周期效果、延迟行为若分散使用 `Time.time` 等判据，易导致隐式耦合、难以统一暂停/回合外逻辑。本项目以 **`ITimingTask`** 统一描述任务标识、状态、优先级与延迟/间隔参数；**`TimingTaskManager`** 负责任务注册与查询；**`TimingTaskScheduler`** 在固定调度节拍上递减计时并触发 **`Execute`**；**`TaskDispatcher`**（及 ECS 下的 **`BasementUnityPump`**）将上述推进与 Unity 的 `Update` / 独立泵送对齐，使 Basement 层时间语义与上层 Gameplay 中其他基于帧的逻辑在文档与调用链上边界清晰。

## 4.5.2.2 任务契约与状态（`ITimingTask`）

```
枚举 TimingTaskState:
    Ready, Running, Completed

枚举 TimingTaskPriority:   // 数值大者优先
    Low = 0, Normal = 1, High = 2

接口 ITimingTask:
    属性 TaskId: 字符串
    属性 State: TimingTaskState
    属性 Priority: TimingTaskPriority
    属性 DelayTime: 浮点     // 秒：初次/周期语义见具体子类
    过程 Execute()
    过程 Cancel()
```

### 抽象基类（`TimingTaskBase`）

```
抽象类型 TimingTaskBase 实现 ITimingTask
    字段 _onCompleted, _onCancelled: Action
    字段 _onFailed: Action<Exception>

构造(taskId?, priority, delayTime):
    TaskId = taskId 或 新生成 Guid
    State = Ready；Priority = priority；DelayTime = delayTime

过程 Execute():
    若 State != Ready 则返回
    State = Running
    尝试:
        调用 ExecuteInternal()          // 子类实现具体行为
        State = Completed；_onCompleted?()
    捕获异常:
        State = Completed；_onFailed?(ex)

过程 Cancel():
    若 State == Completed 则返回
    State = Completed；_onCancelled?()

链式方法 OnCompleted / OnCancelled / OnFailed 用于订阅回调
```

## 4.5.2.3 任务管理器（`TimingTaskManager`）

```
单例 TimingTaskManager
    字段 _tasks: 字典 TaskId -> ITimingTask
    字段 _taskQueue: 列表<ITimingTask>   // 与字典同步，排序用

过程 Initialize():
    幂等标记已初始化

过程 RegisterTask(task):   // 内部：创建方法末尾调用
    加锁
    _tasks[task.TaskId] = task
    _taskQueue.Add(task)
    对 _taskQueue 排序:
        先按 Priority 降序，再按 DelayTime 升序

过程 CreateTimeTriggeredTask(action, delayTime, priority?) -> TimeTriggeredTask:
    task = new TimeTriggeredTask(action, delayTime, priority)
    RegisterTask(task)；返回 task

过程 CreateScheduledTask(action, scheduledTime: DateTime, priority?) -> TimeTriggeredTask:
    delayTime = max(0, (scheduledTime - Now).总秒数)
    task = new TimeTriggeredTask(action, delayTime, priority)
    RegisterTask(task)；返回 task

过程 CreatePeriodicTask(action, interval, repeatCount=-1, priority?) -> PeriodicTask:
    // repeatCount < 0 视为无限次直至外部 Cancel
    task = new PeriodicTask(action, interval, repeatCount, priority)
    RegisterTask(task)；返回 task

过程 CreateConditionalTask(action, condition: Func<bool>, checkInterval=0.1, priority?) -> ConditionalTask:
    task = new ConditionalTask(action, condition, checkInterval, priority)
    RegisterTask(task)；返回 task

过程 CancelTask(taskId) -> bool:
    若命中: task.Cancel()；从 _tasks 与 _taskQueue 移除；返回 true

过程 GetTask(taskId) -> ITimingTask
过程 GetAllTasks() -> 列表   // 快照副本

过程 ClearAllTasks():
    对每个 task 调用 Cancel()；清空字典与队列

过程 ClearCompletedTasks():
    移除所有 State == Completed 的登记项

过程 GetStatistics() -> TimingTaskStatistics:
    统计 Ready / Running / Completed 数量与 TotalCount
```

## 4.5.2.4 任务调度器（`TimingTaskScheduler`）

调度器使用 **`UnityEngine.Time.time`** 与上次推进时刻的差作为原始 `deltaTime`，再按 **`MatchTimeService`** 的对局状态裁剪：**未启用对局**或**已暂停**时视 **`deltaTime = 0`**，使任务倒计时与「局内可流逝时间」一致（与《对局时间模块》叙述对齐）。另有 **`_updateInterval`**（默认约 0.016s）作为调度节拍下限，避免过高频调用。

```
单例 TimingTaskScheduler
    字段 _taskTimers: 字典 ITimingTask -> 剩余秒数
    字段 _activeTasks: 列表<ITimingTask>   // 用于追踪触发明细与 Completed 清理
    字段 _updateInterval: 浮点
    字段 _lastUpdateTime: 浮点

过程 Initialize(): 幂等

过程 Update():
    若 Time.time - _lastUpdateTime < _updateInterval 则返回
    deltaTime = Time.time - _lastUpdateTime
    _lastUpdateTime = Time.time

    match = MatchTimeService.Instance
    若 not match.IsMatchActive 或 match.IsMatchPaused:
        deltaTime = 0

    ProcessTasks(deltaTime)

过程 ProcessTasks(deltaTime):
    加锁
    allTasks = TimingTaskManager.Instance.GetAllTasks()

    对 each task in allTasks:
        若 task.State == Ready:
            若 task 不在 _taskTimers: _taskTimers[task] = task.DelayTime
            _taskTimers[task] -= deltaTime

            若 _taskTimers[task] <= 0:
                若 _activeTasks 不包含 task: _activeTasks.Add(task)
                task.Execute()

                若 task 是 PeriodicTask 且 State 仍为 Ready:
                    _taskTimers[task] = PeriodicTask.Interval
                否则若 task 是 ConditionalTask 且 State 仍为 Ready:
                    _taskTimers[task] = ConditionalTask.CheckInterval
                否则:
                    _taskTimers.Remove(task)

        否则若 task.State == Completed:
            _taskTimers.Remove(task)

    _activeTasks.RemoveAll(t => t.State == Completed)

过程 SetUpdateInterval(interval):
    _updateInterval = max(0.001, interval)
```

> **编入说明：** 上文「到期触发 + 周期/条件重置间隔」对应论文中「延迟执行、周期性触发、按间隔轮询条件」等语义；与工程源码对照时以实现为准（如 `_activeTasks` 仅用于生命周期清理、调度节拍与 `MatchTime` 门控等细节以保持行为一致）。

## 4.5.2.5 具体任务类型

### 一次性延迟（`TimeTriggeredTask`）

```
类型 TimeTriggeredTask : TimingTaskBase
    字段 _action: Action

ExecuteInternal():
    _action()

静态 CreateScheduledTask(action, scheduledTime):
    delay = max(0, scheduledTime - DateTime.Now)
    返回 new TimeTriggeredTask(action, delay, priority)
```

### 周期任务（`PeriodicTask`）

```
类型 PeriodicTask : TimingTaskBase
    字段 _action；字段 _repeatCount；字段 _currentRepeat

// 基类 DelayTime 在此表示 Interval

重写 Execute():
    若 State != Ready 则返回
    State = Running
    尝试:
        _action()；_currentRepeat++
        若 _repeatCount > 0 且 _currentRepeat >= _repeatCount:
            State = Completed；_onCompleted?()
        否则:
            State = Ready        // 等待调度器在下一 Interval 后再次触发
    捕获异常: State = Completed；_onFailed?(ex)

属性 Interval => DelayTime
```

### 条件任务（`ConditionalTask`）

```
类型 ConditionalTask : TimingTaskBase
    字段 _action；字段 _condition: Func<bool>

// 基类 DelayTime 在此表示 CheckInterval

重写 Execute():
    若 State != Ready 则返回
    尝试:
        若 _condition():
            State = Running
            _action()
            State = Completed
            _onCompleted?()
        // 否则保持 Ready，由调度器在下一 CheckInterval 再尝试
    捕获异常: State = Completed；_onFailed?(ex)

属性 CheckInterval => DelayTime
```

## 4.5.2.6 Unity 派发（`TaskDispatcher`）与集中泵送（`BasementUnityPump`）

### 场景组件 `TaskDispatcher`

```
组件 TaskDispatcher : MonoBehaviour

Awake():
    TimingTaskManager.Instance.Initialize()
    TimingTaskScheduler.Instance.Initialize()
    MatchTimeService.Instance.BeginMatch()   // 无显式对局编排时的默认可推进策略

Update():
    若 BasementLoopRouting.IsEcsPumpActive: 返回   // ECS 已接管泵送
    MatchTimeService.Instance.TickUpdate()
    TimingTaskScheduler.Instance.Update()

FixedUpdate():
    若 IsEcsPumpActive: 返回
    MatchTimeService.Instance.TickFixedUpdate()

OnDestroy():
    TimingTaskManager.Instance.ClearAllTasks()

静态 Create() / GetOrCreate(): 在场景中挂载或创建 GameObject("TaskDispatcher")
```

### 静态泵送（与 `BasementPumpEcsSystem` 同序）

```
静态类 BasementUnityPump

过程 PumpMatchTimeUpdate()  => MatchTimeService.Instance.TickUpdate()
过程 PumpMatchTimeFixed()   => MatchTimeService.Instance.TickFixedUpdate()
过程 PumpTimingTasks()      => TimingTaskScheduler.Instance.Update()
过程 PumpGameEvents()       => GameEventScheduler.Instance.Update()

过程 PumpUpdate():
    PumpMatchTimeUpdate()
    PumpTimingTasks()
    PumpGameEvents()

过程 PumpFixedUpdate():
    PumpMatchTimeFixed()
```

### 运行时引导（`BasementRuntimeBootstrap`，节选）

## 4.5.2.7 扩展能力：批量处理与任务池（工程内已实现，可与调度组合使用）

### `BatchTaskProcessor`

```
类型 BatchTaskProcessor
    字段 _batch: 列表<ITimingTask>
    字段 _batchSize, _batchInterval, _lastBatchTime

过程 AddTask(task):
    _batch.Add(task)
    若 _batch.Count >= _batchSize 或 Time.time - _lastBatchTime >= _batchInterval:
        ProcessBatch()

过程 ProcessBatch():
    对 _batch 中每项调用 Execute()；清空 _batch；更新 _lastBatchTime

过程 ForceProcess(): 立即 ProcessBatch()

过程 SetBatchSize / SetBatchInterval: 写入下限保护后的参数
```

### `TaskPool<T>`（`T : ITimingTask, new()`）

```
类型 TaskPool<T>
    字段 _pool: 栈<T>；锁

过程 Get() -> T: 栈非空则 Pop，否则 new T()
过程 Return(task): 若非空则 Push           // 配合业务在 Return 前重置任务字段可复用
过程 Clear()
```

> **组合方式（编入正文时可写为既定实践）：** 高频创建短生命周期任务时，可由工厂从 **`TaskPool`** 取出实例、配置后 **`RegisterTask`**；到期 **`Cancel`** 或 **`ClearCompletedTasks`** 后 **`Return`** 入池。批量 **`Execute`** 可由 **`BatchTaskProcessor`** 聚合适用于「允许同一逻辑帧内稍后统一执行」的路径，与调度器的「按时间到期」互为补充。

## 业务侧使用范式（注册—推进—取消）

```
// 注册（管理器侧统一登记，调度器自动推进）
TimingTaskManager.Instance.CreateTimeTriggeredTask(() => { /* 延迟逻辑 */ }, delaySeconds: 2f, priority: High)
TimingTaskManager.Instance.CreatePeriodicTask(() => { /* 每 interval 秒 */ }, interval: 1f, repeatCount: 10)
TimingTaskManager.Instance.CreateConditionalTask(() => { /* 条件达成 */ }, () => player.IsGrounded, checkInterval: 0.1f)

// 取消
TimingTaskManager.Instance.CancelTask(taskId)

// 场景卸载或局终
TimingTaskManager.Instance.ClearAllTasks()
```
