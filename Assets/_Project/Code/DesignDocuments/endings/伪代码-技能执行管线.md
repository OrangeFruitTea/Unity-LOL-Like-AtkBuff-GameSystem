# 5.5.1 技能执行管线逻辑 — 伪代码示例

下文与 `Assets/_Project/Code/Scripts/Gameplay/Skill/`（`Context`、`Config`、`Runtime`、`Ecs`、`Buff`、`Conditions`、`Targeting`）及 `Gameplay.Runtime.GameplaySystemsBootstrap` 等**工程实现对应**，采用**伪代码**便于写入论文 §5.5.1 与表驱动管线叙述；**OnEvent** 类步骤在设计语义上保留「事件到达后执行」，文末注明与当前 `ProcessEventStub` 行为的对照。

---

## 5.5.1.1 设计意图

技能释放进入逻辑层后，将「本次施法」整理为 **`SkillCastContext`**（施法者、主/副目标、等级、施法起点时间等），与静态 **`SkillDefinition.Steps`** 解耦。**`PipelineScheduleBuilder`** 把表驱动步骤折叠为 **按 `Time.time` 相对施法起点的定时批次**，并与 **条件触发**、**（可选）事件触发** 步骤并列维护。**`SkillCastPipelineSystem`** 在 **`EcsWorld.Update`** 中与 **`UpdateOrder = 50`** 的其他系统同节拍推进，每帧检查条件与定时批次，满足则经 **`BuffApplyService`** 把 Buff 挂到解析出的目标上，从而把复杂性收拢到 **JSON/配置 + 注册表**，减少手写 `if-else` 编排。**`SkillCooldownTracker`** 在门面 **`TryBeginCast`** 成功路径上写入冷却；管线与 **`ImpactSystem`**、**`BuffTickEcsSystem`**、**`UnitVitalitySystem`** 等通过固定 **`UpdateOrder`** 顺序衔接。

## 5.5.1.2 施法上下文（`SkillCastContext`）

```
类型 SkillCastContext（引用语义，单次施法一实例）

构造(caster: EntityBase):
    Caster = caster
    SecondaryTargets = 空列表

属性:
    Caster: EntityBase（只读）
    PrimaryTarget: EntityBase
    SecondaryTargets: 列表<EntityBase>
    SkillLevel: int，默认 1
    RandomSeed: int
    CastStartedUnityTime: float    // 由 SkillExecutionFacade 在 TryBeginCast 成功前设为 Time.time

过程 Validate(out error) -> bool:
    若 Caster 为空 → error；返回 false
    若 Caster 非合法 Buff/技能宿主（EntityEcsBridge.IsValidBuffTarget）→ error；返回 false
    返回 true
```

## 5.5.1.3 技能定义与步骤（`SkillDefinition` / `BuffApplicationStepDefinition`）

### 技能定义

```
类型 SkillDefinition:
    SkillId: int
    DisplayName: 字符串
    MaxLevel: int
    CooldownSeconds: float
    RequiresTarget: bool       // true 时 Context 必须带 PrimaryTarget
    CastRange: float
    Steps: 列表<BuffApplicationStepDefinition>
```

### 触发种类（`BuffApplicationTriggerKind`）

```
枚举 BuffApplicationTriggerKind:
    Immediate              // t=0 与施法起点对齐
    AfterDelay             // t = DelaySecondsFromCastStart（≥0）
    OnCondition            // 每帧由 SkillConditionRegistry 求值，真则执行一步
    OnEvent                // 设计语义：外部事件 id 匹配后执行（见 §5.5.1.6 实现注）
```

### 单步 Buff 施加描述

```
类型 BuffApplicationStepDefinition:
    StepId: 字符串
    BuffId: int
    TargetSelector: SkillTargetSelectorKind   // Caster | PrimaryTarget | SecondaryTargets
    BuffLevel: uint
    LevelScalingPerSkillLevel: uint            // 每级技能叠加的 Buff 等级增量
    DurationOverride: 可空 float
    CustomArgs: 列表<object>
    TriggerKind: BuffApplicationTriggerKind
    DelaySecondsFromCastStart: float           // AfterDelay 用
    ParallelGroup: int                         // 表字段保留，定时批次按时间归并
    ConditionId / ConditionParam: 条件步参数
    EventId: 字符串                            // OnEvent 用
```

### 目标选择器（`DefaultTargetResolver`）

```
过程 Resolve(context, selector) -> 只读列表<EntityBase>:
    Caster           -> [context.Caster] 或空
    PrimaryTarget    -> [context.PrimaryTarget] 或空
    SecondaryTargets -> context.SecondaryTargets 或空
    默认             -> 空数组
```

### Buff 等级解析（`SkillParameterResolver`）

```
过程 ResolveBuffLevel(step, skillLevel) -> uint:
    lv = max(1, skillLevel)
    返回 step.BuffLevel + max(0, lv - 1) * step.LevelScalingPerSkillLevel
```

## 5.5.1.4 管线排期（`PipelineScheduleBuilder`）

### 定时批次（Immediate + AfterDelay）

```
类型 ExecutionBatch:
    FireTimeFromCastStart: float    // 相对施法起点的 Unity 时间偏移
    StepIndices: 列表<int>         // 同刻触发的多步可并入一批

过程 BuildTimedBatches(definition) -> 列表<ExecutionBatch>:
    若 Steps 为空 → 返回空列表

    entries = 空列表
    对 i, step in definition.Steps:
        若 step.TriggerKind == Immediate:
            entries 加入 (t=0, idx=i)
        若 step.TriggerKind == AfterDelay:
            entries 加入 (t=max(0, step.DelaySecondsFromCastStart), idx=i)
        // OnCondition / OnEvent 不进入定时批次

    按 (t, idx) 排序
    将 **连续相同 t**（Approx 相等）的条目合并为一个 ExecutionBatch（一步骤列表）
    返回批次列表（时间非减）
```

### 条件步与事件步索引

```
过程 CollectConditionStepIndices(def) -> 列表<int>:
    所有 TriggerKind == OnCondition 的步骤下标

过程 CollectEventStepIndices(def) -> 列表<int>:
    所有 TriggerKind == OnEvent 的步骤下标
```

## 5.5.1.5 条件注册（`SkillConditionRegistry`）

```
静态类 SkillConditionRegistry
    字典 conditionId -> Func<SkillCastContext, BuffApplicationStepDefinition, bool>

过程 Register(conditionId, fn)
过程 TryEvaluate(conditionId, context, step) -> bool:
    若 conditionId 为空 → 视为 true
    若无注册 → false
    否则安全调用 fn(context, step)

过程 RegisterBuiltInDefaults():
    例：entity_hp_below_ratio
        读 PrimaryTarget 的 CrtHp / HpLimit，若 hp/max <= step.ConditionParam 则真
```

> **启动时**：`GameplaySystemsBootstrap` 在注册 ECS 系统前调用 **`RegisterBuiltInDefaults()`**，保证条件步可解析。

## 5.5.1.6 技能目录与门面入口（`SkillCatalog` / `SkillExecutionFacade`）

```
静态类 SkillCatalog（由 SkillDataLoader / EcsWorld 从 StreamingAssets 等填充）
    TryGet(skillId, out SkillDefinition)

静态类 SkillExecutionFacade

过程 TryBeginCast(skillId, context, respectCooldown=true, out error) -> bool:
    若 not SkillCatalog.TryGet(skillId, out def) → error；返回 false
    若 not context.Validate(out error) → 返回 false
    若 def.RequiresTarget 且 context.PrimaryTarget == null → error；返回 false

    ecsId = context.Caster.BoundEcsEntity.Id
    若 ecsId == 0 → error；返回 false

    若 respectCooldown 且 not SkillCooldownTracker.IsReady(ecsId, skillId) → error；返回 false

    context.CastStartedUnityTime = Time.time

    system = EcsWorld.GetEcsSystem<SkillCastPipelineSystem>()
    若 system == null → error；返回 false

    若 not system.TryBegin(def, context, out error) → 返回 false

    若 respectCooldown:
        SkillCooldownTracker.NotifyCast(ecsId, skillId, def.CooldownSeconds)

    context.Caster.GetComponent<UnitAnimDrv>()?.NotifySkillCastStarted()
    返回 true

过程 CancelActivePipeline(caster):
    GetEcsSystem<SkillCastPipelineSystem>()?.CancelForCaster(caster)
```

## 5.5.1.7 冷却（`SkillCooldownTracker`）

```
静态类 SkillCooldownTracker（Gameplay.Skill.Runtime）
    字典 (casterEcsId, skillId) -> NextReadyUnityTime

过程 IsReady(casterEcsId, skillId) -> bool:
    若无记录 → true
    否则返回 Time.time >= NextReadyUnityTime

过程 NotifyCast(casterEcsId, skillId, cooldownSeconds):
    若 cooldownSeconds <= 0 → 返回
    NextReadyUnityTime[(casterEcsId, skillId)] = Time.time + cooldownSeconds

过程 Clear(): 清空字典（切场景或测试用）
```

## 5.5.1.8 管线系统（`SkillCastPipelineSystem`，`UpdateOrder = 50`）

### 会话结构

```
类型 SkillCastSession（系统内部）:
    CastInstanceId: 字符串
    Definition: SkillDefinition
    Context: SkillCastContext
    TimedBatches: 列表<ExecutionBatch>
    NextBatchIndex: int
    PendingConditionSteps: 集合<int>     // OnCondition 步骤下标
    PendingEventSteps: 集合<int>          // OnEvent 步骤下标
    Cancelled: bool
    LoggedEventStub: bool                // 避免重复警告
```

### TryBegin

```
过程 TryBegin(definition, context, out error) -> bool:
    校验 definition、context.Caster、caster.BoundEcsEntity.Id != 0
    若 _sessions 已含该施法者 ecsId → error（每位施法者同时仅一条活跃管线）
    session = new SkillCastSession {
        CastInstanceId = 新 Guid,
        TimedBatches = PipelineScheduleBuilder.BuildTimedBatches(definition),
        NextBatchIndex = 0,
        PendingConditionSteps = CollectConditionStepIndices(definition),
        PendingEventSteps = CollectEventStepIndices(definition),
        Definition = definition,
        Context = context
    }
    _sessions[casterEcsId] = session
    SetPipelineComponentFlag(caster, HasActiveCast = true)
    返回 true
```

### Update（与 ECS 同帧）

```
过程 Update():
    now = Time.time
    done = 空列表

    对每个 session in _sessions（按施法者 ecsId）:
        若 session.Cancelled → 加入 done；continue
        若 not IsContextAlive(session.Context) → 加入 done；continue
            // IsContextAlive: Caster 存在且仍为合法 Buff 宿主

        ProcessConditionSteps(session, now)   // 见下
        ProcessTimedBatches(session, now)   // 见下
        ProcessEventStub(session)           // 见下 · 实现注

        若 IsSessionComplete(session) → 加入 done

    对每个 id in done: EndSession(id)
        // EndSession: 移除会话； caster 上 SkillPipelineRuntimeComponent.HasActiveCast = false
```

### 条件步

```
过程 ProcessConditionSteps(session, now):
    若 PendingConditionSteps 为空 → 返回
    fired = 空列表
    对每个 idx in PendingConditionSteps:
        step = Definition.Steps[idx]
        若 not SkillConditionRegistry.TryEvaluate(step.ConditionId, Context, step) → continue
        ExecuteStep(session, idx)
        fired.Add(idx)
    从 PendingConditionSteps 移除 fired 中各项
```

### 定时批次

```
过程 ProcessTimedBatches(session, now):
    t0 = Context.CastStartedUnityTime
    当 NextBatchIndex < TimedBatches.Count:
        batch = TimedBatches[NextBatchIndex]
        若 now < t0 + batch.FireTimeFromCastStart → break
        对每个 stepIndex in batch.StepIndices:
            ExecuteStep(session, stepIndex)
        NextBatchIndex++
```

### 事件步（设计语义 vs 实现注）

```
过程 ProcessEventStub(session):
    // 设计叙述（论文可写）：订阅 gameplay 事件总线或战斗事件，当 eventId 与步骤 EventId 匹配时 ExecuteStep
    //
    // 当前工程实现：若存在 PendingEventSteps 且未打过桩日志，则打一次 Warning（OnEvent 未实现订阅）、
    //               清空 PendingEventSteps，使会话仍可按「无待定事件步」结束。
    若 PendingEventSteps 为空 或 LoggedEventStub → 返回
    LoggedEventStub = true；打日志；PendingEventSteps.Clear()
```

### 会话完成判定

```
过程 IsSessionComplete(session) -> bool:
    timedDone = NextBatchIndex >= TimedBatches.Count
    condDone = PendingConditionSteps 为空
    eventDone = PendingEventSteps 为空    // 清空后即为 true
    返回 timedDone && condDone && eventDone
```

### 执行单步 → Buff

```
过程 ExecuteStep(session, stepIndex):
    step = Definition.Steps[stepIndex]
    ctx = Context
    targets = TargetResolver.Resolve(ctx, step.TargetSelector)
    level = SkillParameterResolver.ResolveBuffLevel(step, ctx.SkillLevel)

    对每个 target in targets:
        若 target 为空或非法宿主 → continue
        BuffApplyService.TryApplyStep(step, target, ctx.Caster, level, modifiers=null, out err)
        若失败 → 打 Warning（含 stepId、buffId、err）
```

### 取消

```
过程 CancelForCaster(caster):
    若 _sessions 含 caster.BoundEcsEntity.Id → 对应 session.Cancelled = true
    // 下一帧 Update 将 EndSession 并清 HasActiveCast
```

## 5.5.1.9 Buff 施加服务（`BuffApplyService`）

```
静态类 BuffApplyService

过程 TryApplyStep(step, target, provider, resolvedBuffLevel, modifiers, out error) -> bool:
    req = BuffApplyRequest.FromStep(step, target, provider, resolvedBuffLevel)
    返回 TryApply(req, modifiers, out error)

过程 TryApply(request, modifiers, out error) -> bool:
    校验 target、provider、合法宿主
    （可选）modifiers 逐个 Modify(request)
    BuffTypeRegistry.TryGetFactory(request.BuffId, IBuffFactory)
    factory.Apply(target, provider, level, durationOverride, customArgsTail)
    返回 true；异常则 error
```

> ** opcode / Impact**：具体 Buff 工厂内可调度 **`ImpactManager.CreateImpactEvent`** 等，与 §5.4 伤害链汇合；管线层不直接分支伤害类型。

## 5.5.1.11 与 ECS 主循环的衔接

- **`SkillCastPipelineSystem.UpdateOrder => 50`**，由 **`EcsWorld.Update`** 按序调用；同帧内早于它的系统含 **`ImpactSystem`（31）**、**`UnitVitalitySystem`（32）**、**`BuffTickEcsSystem`** 等，便于「本帧已扣血 / Buff tick」与「本帧技能段」在文档上区分先后。
- **`GameplaySystemsBootstrap.AfterSceneLoad`**：`BuffManager.EnableEcsDriving()` → **`SkillConditionRegistry.RegisterBuiltInDefaults()`** → **`TryAdd`** 各 **`IEcsSystem`**（含 **`SkillCastPipelineSystem`**）。
