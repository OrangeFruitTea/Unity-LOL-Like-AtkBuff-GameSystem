## 5.4.2.1 设计意图

瞬时伤害、治疗、临时属性修正若散落在各模块直接改写数值，易产生隐式耦合与口径不一致。本项目将一次「来源对目标施加的数值作用」抽象为 **Impact 事件实体**：携带 **Source、Target、TargetAttribute、OperationType、ImpactType、SourceType**（及 **`ImpactValueComponent.BaseValue`** 等），由 **`ImpactSystem`** 在 ECS 更新阶段**统一消费**，经命中判定、暴击、**目标侧修饰器**与防减伤合成后，写入 **`EntityDataComponent`**。**战斗黑板**记录承伤来源；**受击表现**经 **`HitFxRelay`** 单向触发；**击倒与死亡播报**交给 **`UnitVitalitySystem`**（`UpdateOrder` 大于 `ImpactSystem`）等后续系统，保持 Impact 单次、数据层职责纯粹。

## 5.4.2.3 事件创建（`ImpactManager.CreateImpactEvent`）

```
类 ImpactManager（绑定 EcsWorld.CombatImpacts 单例入口）

过程 CreateImpactEvent(source, target, targetAttribute, value,
                            operationType=Add, impactType=Physical, sourceType=NormalAtk) -> EcsEntity:

    eventEntity = world.CreateEntity()

    ev = new ImpactEventComponent.InitializeDefaults()
    ev.Source = source
    ev.Target = target
    ev.TargetAttribute = targetAttribute
    ev.OperationType = operationType
    ev.ImpactType = impactType
    ev.SourceType = sourceType
    eventEntity.AddComponent(ev)

    val = new ImpactValueComponent.InitializeDefaults()
    val.BaseValue = value
    val.Type = impactType
    eventEntity.AddComponent(val)

    返回 eventEntity

过程 AddModifier(target, modifier):
    确保 target 上有 ImpactModifierComponent
    Modifiers.Add(modifier)；SetComponent

过程 RemoveModifier(target, source)、ClearModifiers(target):   // 按 Source 或全部清除
```

### 普攻示例（与 `DefaultCombatImpactDispatch` 一致）

```
rawAtk = 攻击者.EntityData.GetData(AtkAD)

CombatImpacts.CreateImpactEvent(
    攻击者.EcsEntity,
    受击者.EcsEntity,
    TargetAttribute.Hp,
    rawAtk,
    ImpactOperationType.Subtract,
    ImpactType.Physical,
    ImpactSourceType.NormalAtk)
```

治疗、临时加攻击等：**同一接口**，仅替换 `TargetAttribute`（如 `Hp` + **`Add`**）、`ImpactType`（如治疗常配合 `Magical` 等口径）与 `SourceType`（如 **`Skill`** / **`Buff`**）。

## 5.4.2.4 `ImpactSystem` 主循环（`UpdateOrder = 31`）

```
类 ImpactSystem 实现 IEcsSystem
    字段 _processedEvents: 集合<EventId>   // 去重与短闭环

过程 Update():
    eventEntities = world.GetEntitiesWithComponent<ImpactEventComponent>().ToList()   // 快照，避免遍历时销毁集合失真

    对 eventEntities 中每个 eventEntity:
        impact = eventEntity.GetComponent<ImpactEventComponent>()

        若 impact.IsProcessed 或 EventId 已见于 _processedEvents: 跳过

        若 Source/Target 无效 或 Target 无 EntityDataComponent 或 PerformHitCheck != Hit:
            标记 IsProcessed、写回 ImpactEventComponent、记入 _processedEvents
            continue   // 与成功路径不同：当前实现对失败分支不 Destroy 事件实体

        baseValue = eventEntity 若有 ImpactValueComponent 则取其 BaseValue，否则 0

        impact.IsCritical = CheckCritical(impact)    // 如来源 EntityData.CriticalRate 抽签
        若 IsCritical: baseValue *= GetCriticalMultiplier(impact)

        finalValue = CalculateFinalValue(impact, baseValue)

        ApplyImpactToTarget(impact, finalValue)

        impact.IsProcessed = true；SetComponent；_processedEvents.Add(EventId)
        world.DestroyEntity(eventEntity)             // 临时事件实体本轮结束销毁

    CleanupProcessedEvents()    // 如集合过大则清空，防长期增长

过程 PerformHitCheck(impact) -> HitResult:
    // 可扩展闪避、抵抗；当前实现恒为 Hit
    返回 Hit
```

## 5.4.2.5 最终数值合成（修饰器 + 防类型）

```
过程 CalculateFinalValue(impact, baseValue) -> float:
    v = baseValue

    若 impact.Target 含 ImpactModifierComponent:
        modifiers = 按 Modifier.Priority 降序排列
        对每个 m:
            若 m.IsPercentage: v *= (1 + m.Value)
            否则: v += m.Value

    若 impact.ImpactType == Physical 且 Target 有 EntityData:
        defence = Target.GetData(DefenceAD)
        v *= (100 / (100 + defence))           // 工程内简化护甲公式

    否则若 impact.ImpactType == Magical:
        mr = Target.GetData(DefenceAP)
        v *= (100 / (100 + mr))

    // ImpactType.True：跳过防修正

    返回 v
```

## 5.4.2.6 应用到 `EntityDataComponent`（`ApplyImpactToTarget`）

```
过程 ApplyImpactToTarget(impact, finalValue):
    targetData = impact.Target.GetComponent<EntityDataComponent>()

    根据 impact.TargetAttribute 分支:
        Hp    -> ApplyHpChange(targetData, finalValue, OperationType)      // [0, HpLimit] Clamp
        Mp    -> ApplyMpChange（同理 Clamp 到 MpLimit）
        AtkAD, AtkAP, DefenceAD, ... Core 键 -> ApplyCoreAttributeChange(..., EntityBaseDataCore 对应键, ...)
        其余 Bonus 键（如 HpRecoverPerSecond, PenAD, LifeSteal, MoveSpeed, CriticalDamage ...）
            -> ApplyBonusAttributeChange(..., EntityBaseData 对应键, ...)

    通用算术（Add / Subtract / Override）后，对 HP/MP 做上下界约束；属性变更工程内对下限常 Clamp 到 0

    // —— 承伤黑板（还击、击杀归因）——
    若 TargetAttribute == Hp 且 OperationType == Subtract 且 finalValue > 0
       且 Source 有效 且 Target 含 CombatBoardLiteComponent:
        board.LastDamageFromEntityId = Source.Id
        Target.SetComponent(board)

    // —— 单向表现（不反向依赖玩法规则）——
    若 TargetAttribute == Hp 且 OperationType == Subtract 且 finalValue > 0:
        HitFxRelay.RaiseHpDamaged(Target, finalValue)   // 内部经 EntityEcsLinkRegistry → UnitAnimDrv.NotifyDamaged

    Target.SetComponent(targetData)
```

## 5.4.2.7 与 `UnitVitalitySystem` 及后续链路的衔接（`UpdateOrder = 32`）

```
类 UnitVitalitySystem
    字段 _deathAnnounced: 集合<实体Id>

过程 Update():
    对每个含 EntityDataComponent 的 ecs:
        若 CrtHp > 极小正阈值:
            _deathAnnounced.Remove(ecs.Id)；continue

        若无 CombatBoardLiteComponent: continue

        board = ecs.GetComponent<CombatBoardLiteComponent>()
        若 KillerEntityId == 0:
            board.KillerEntityId = board.LastDamageFromEntityId    // 归因补齐
            ecs.SetComponent(board)

        若 ecs.Id 首次加入 _deathAnnounced:
            CombatUnitDeathRelay.AnnounceFirstDeath(ecs, board.KillerEntityId)
            // 经济、全局事件总线、UI 等由 Relay / 订阅方续接，不在 ImpactSystem 内展开
```

**读图要点**：**Impact** 只完成**一次**数值写回、黑板与本地受击提示；**是否击倒、死亡是否已播报**由 **Vitality** 与 **Relay** 在**同一帧稍后**或**后续帧**处理，避免在 Impact 内堆叠全链路分支。
