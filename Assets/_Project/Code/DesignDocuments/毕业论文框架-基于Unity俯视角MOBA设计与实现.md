# 毕业论文框架：基于 Unity 的俯视角 MOBA 游戏设计与实现

本文档对齐《电子科技大学学士学位论文撰写范例》（绪论 → 分章正文含「本章小结」→ **总结与展望**）。全文技术叙事与仓库一致：**局域网侧采用 Mirror**；结构与写作重心见下文「撰写侧重点」，便于与各章提纲配套使用。

---

## 〇、撰写侧重点（动笔前可看）

本文建议将笔墨集中在 **需求与总体架构**、**基础设施与网络化适配的工程化描述**、**局内子系统的分层设计说明**、**交互与界面设计及图示**。绪论与技术综述可走教材与文献口吻；网络相关章节侧重 Mirror 基本概念与工程中 **façade / 会话层** 的职责划分。**表现层章节**可多配流程图、信息架构图、界面草图或编辑器截图以增强可读性。各章的具体技术论据与图示来源以 `DesignDocuments/` 及工程脚本为主。

| 优先级 | 建议展开方式 |
|--------|----------------|
| 需求与分层架构 | 按「对战核心—交互辅助—信息服务」模块化叙述，辅以架构图与模块职责表 |
| Mirror 与网络文档 | 概念性介绍 + `NetworkSyncService.md`、`网络同步服务脚本接口设计文档.md` 与 `GameplayNetworkManager` 等工程落点 |
| 局内系统设计 | `工程模块与命名空间-现行实现总览.md`：ECS、Impact、Targeting、Buff/技能管线、兵线/野区/塔等，用**系统设计、数据流与控制流**句式叙述 |
| UI 与视听 | 操作流程、控件布局原则、图示与案例分析 |

正文表述以 **设计方案、工作流程、关键技术点**为主干；与历史材料中网络方案表述不一致时，在 **技术与选型章节**以一自然段说明工程采用的 Mirror 方案即可。
