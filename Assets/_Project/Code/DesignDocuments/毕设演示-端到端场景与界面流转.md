# 毕设演示：端到端场景与界面流转（单机）

| 项 | 内容 |
|----|------|
| 文档版本 | 1.0 |
| 性质 | **演示专用冻结说明**：把「从开机到胜负再到可再来一局」落到 **场景名 + 迁移触发 + Phase**；实现与答辩录像据此对齐 |
| **演示范围** | **仅单机**：不依赖局域网、不依赖 Mirror 会话；网络方案仍以仓库 `NetworkSyncService.md` 等为扩展阅读，**不纳入本演示验收** |
| 关联文档 | `MOBA对局信息与胜负流程模块设计文档.md`（§4.3 演示冻结、`BeginMatch`/`EndMatch`）、`对局时间模块设计文档.md`、`局内输入与NewInputSystem接轨设计.md` |

---

## 1. 目标与验收一句话

玩家在 **主菜单** 进入 **一局单机对战**，达成 **冻结的胜利条件**（摧毁敌方基地，见对局文档 §4.3）后，看到 **结算界面**，可 **返回主菜单** 再开一局；全程 **无需** 第二台机器或联网。

---

## 2. 场景级流转（推荐最小集合）

工程可按现有目录微调命名；下表为 **逻辑角色**，不要求文件名一字不差。

| 场景（逻辑名） | 职责 |
|----------------|------|
| **`MainMenu`** | 标题 / 主菜单：**开始演示对局**、**退出游戏**；可选 **设置**（音量等）。 |
| **`Arena_Demo`**（或当前 Sample 战斗场景） | **唯一局内场景**：生成地图、玩家单位、敌方单位/AI、**基地等可摧毁目标**；挂载 **MatchFlow**（或等价编排器）。 |
| **结算表现** | **推荐**：仍在 `Arena_Demo` 内 **全屏 Panel 叠加**（不卸载场景），避免异步加载打断演示；**备选**：独立 `Settlement` 场景（需明确谁 `LoadScene`）。下文默认 **叠加 Panel**。 |

**不提供单独 Loading 场景亦可**：用 **Phase = Loading** 覆盖「异步加载 + 黑屏/进度条」的若干帧；答辩说明即可。

---

## 3. Phase ↔ 界面 / 操作（单机）

| Match Phase | 玩家可见 | 输入与说明 |
|-------------|----------|------------|
| **None** | 主菜单 | 「开始」→ 触发 **EnterMatch**。 |
| **Loading** | 全屏遮罩或简短黑屏（可选） | 加载 `Arena_Demo`（若尚未载入）；生成单位与目标；完成后 → **Countdown**。 |
| **Countdown** | 可选 3/2/1 文案 | 结束瞬间 → **Playing**，并调用 **`BeginMatch()`**（对局时间）。 |
| **Playing** | 局内 HUD + 操控 | 战斗；**胜负判定仅在此阶段**；**Esc** → **Paused**（见输入文档）。 |
| **Paused** | 暂停菜单 | **继续** / **返回主菜单**（中止本局记 `Aborted` 或等价处理，需在实现里固定一种）。 |
| **Settling** | **结算 Panel** | 展示 `MatchSummary` 最小字段；**禁止** 战斗输入；提供 **返回主菜单**。 |
| **Finished** | 关闭结算后 | **`EndMatch()`**；`LoadScene(MainMenu)` **或** 仅 SetActive 菜单 UI（二选一写死）。 |

与 **`IMatchTimeControl`** 的精确挂钩见 **`MOBA对局信息与胜负流程模块设计文档.md` §4.3 Checklist**。

---

## 4. 关键迁移触发（实现核对表）

| # | 触发条件 | 动作 |
|---|----------|------|
| 1 | 主菜单点击「开始」 | `EnterMatch`：`Loading`；载入/激活 `Arena_Demo`。 |
| 2 | 场景就绪且单位生成完毕 | `Loading → Countdown`（可 `Countdown` 长度为 0）。 |
| 3 | Countdown 结束 | `Playing` + **`BeginMatch()`**。 |
| 4 | **`MatchVictoryDecided`**（单机本地裁定） | `Playing → Settling`；弹出结算 Panel；组装 **`MatchSummary`** → **`MatchRecordBuilt`**。 |
| 5 | 结算上点击「返回主菜单」 | **`EndMatch()`**；清理本局引用（含 ECS/生成物策略由工程约定）；`Finished` → 回 **`MainMenu`**。 |
| 6 | Loading 致命失败 | `Aborted`；Toast 或对话框 → 回 **`MainMenu`**。 |

---

## 5. 输入与演示路径（交叉引用）

局内 **移动、普攻、技能、暂停** 的设备映射与 Binder 约定见 **`局内输入与NewInputSystem接轨设计.md`**。

本演示文档仅要求：**评委可操作的角色在 Playing 阶段能完成「打到摧毁敌方基地」所需的最小操作**；具体键位以实现为准，答辩材料中附 **半页操作说明** 即可。

---

## 6. 与非演示模块的边界

| 不做（本演示不验收） | 说明 |
|----------------------|------|
| 局域网 Host/Client | 不启动 Mirror 房间流程 |
| 战绩 JSON / 生涯 | 可选；闭环验收以 **内存结算 + UI** 为准 |
| 商店必经流程 | 除非胜利条件绑定经济，否则可不强制 |
