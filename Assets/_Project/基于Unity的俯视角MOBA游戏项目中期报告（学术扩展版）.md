## 1. 引言

### 1.1 项目背景

随着游戏产业的快速发展，MOBA（Multiplayer Online Battle Arena）游戏已成为当下最受欢迎的游戏类型之一。这类游戏以其独特的竞技性、策略性和社交性吸引了大量玩家。然而，传统的MOBA游戏通常具有较高的硬件要求和复杂的操作机制，限制了部分玩家的参与。

基于此背景，本项目旨在开发一款轻量化的俯视角MOBA游戏，通过Unity引擎的强大功能，实现经典MOBA核心玩法的同时，降低硬件要求和操作门槛，让更多玩家能够体验MOBA游戏的乐趣。此外，项目还注重局域网联机功能的实现，满足玩家线下社交互动的需求。

### 1.2 项目目标

本项目的主要目标包括：

1. **功能目标**：实现经典MOBA核心玩法，包括对线补兵、英雄技能释放、团战对抗、资源争夺等关键玩法；支持单机游玩与局域网联机对局；提供角色信息浏览与历史对局分析功能。

2. **技术目标**：采用分层+模块化+MVC核心设计的架构方案，确保系统的高内聚、低耦合、易扩展、可维护；实现高效的网络同步机制，确保多端数据一致；优化资源管理与性能，确保游戏运行流畅。

3. **质量目标**：确保游戏的稳定性与可靠性，提供良好的用户体验；符合软件工程的质量标准与规范，确保代码的可读性、可维护性与可扩展性。

### 1.3 研究方法

本项目采用以下研究方法：

1. **文献研究**：查阅相关书籍、论文和技术文档，了解MOBA游戏的设计理念与实现方案，掌握Unity引擎的核心功能与最佳实践。

2. **系统分析与设计**：采用软件工程的系统分析方法，明确项目的需求与功能，设计合理的系统架构与模块划分。

3. **实验与测试**：通过实验验证技术方案的可行性，通过测试确保系统的质量与性能。

4. **迭代开发**：采用敏捷开发方法，迭代式开发，快速响应用户需求，不断优化与完善系统。

## 2. 毕业设计的进展情况

### 2.1 课题工作完成情况

在项目中期，我们完成了以下核心工作：

#### 2.1.1 系统架构设计与实现

**架构设计**：采用分层+模块化+MVC核心设计的架构方案，将系统划分为四个层级：

1. **基础设施层**：提供引擎适配、资源管理、配置解析、日志调试等通用能力，作为系统运行的基础支撑。
2. **核心业务层**：封装网络同步、数据存储、事件调度、时间管理等核心业务，为业务逻辑提供标准化接口。
3. **业务逻辑层**：MVC模式与ECS架构混合使用，拆分核心玩法模块，承载角色、对战规则、经济系统等核心逻辑。
4. **交互表现层**：负责处理用户输入、界面显示、音效特效组织与表现，直接处理用户的操作和感官体验。

**实现情况**：

- **基础设施层**：实现了引擎适配模块，封装了Unity引擎核心API，形成标准化的接口库，支持引擎替换和单元测试；实现了资源管理模块，基于资源池技术设计对资源生命周期的管控，支持多种加载策略；实现了配置解析模块，构建了通用配置文件解析框架，支持多种配置格式和热更新；实现了日志与调试模块，设计了多级别日志记录系统。

- **核心业务层**：实现了网络同步服务，基于Photon PUN 2提供局域网联机核心能力；实现了数据存储服务，负责本地数据的存取；实现了事件调度服务，设计了全局事件中心；实现了时间管理服务，封装了时序任务调度系统。

- **业务逻辑层**：实现了角色系统模块，封装了角色核心数据与状态管理；实现了对战规则模块，封装了对局内的交互逻辑；实现了联机对战模块，调用网络同步服务实现局域网联机功能；实现了信息管理模块，加载并缓存角色的配置数据。

- **交互表现层**：目前处于设计阶段，计划在后续开发中实现用户输入处理、界面显示、音效特效组织与表现等功能。

#### 2.1.2 核心系统开发

**Buff系统**：
- 实现了完整的Buff管理系统，包括Buff的添加、移除、更新、冲突处理等功能
- 采用对象池技术对Buff对象进行缓存与复用，减少内存分配与GC开销
- 实现了三种Buff冲突解决策略：合并（Combine）、分离（Separate）、覆盖（Cover）
- 通过协程实现Buff的定时更新，确保Buff效果的实时计算与应用
- 实现了垃圾回收机制，定期清理无用的Buff对象与引用

**实体系统**：
- 实现了基于ECS架构的实体管理系统，支持实体的创建、销毁、组件管理等功能
- 将实体的属性与行为分离为组件，通过系统对组件进行处理，提高代码的可维护性与可扩展性
- 通过组件数据驱动游戏逻辑，减少代码耦合，提高系统的灵活性



**网络同步系统**：
- 基于Photon PUN 2实现了局域网联机功能，支持房间管理、玩家状态同步、对战指令传输等
- 采用状态同步+事件同步的混合策略，确保多端数据一致
- 实现了断连重连机制，确保玩家在网络波动后能够恢复游戏状态

#### 2.1.3 功能模块开发

**角色系统**：
- 实现了角色的属性管理、状态管理、技能释放等功能
- 每个角色具备独立的属性与技能，支持技能的等级提升与效果叠加
- 实现了角色的可视化表现、状态显示与更新

**技能系统**：
- 实现了技能的释放触发、弹道运动、命中判定、效果结算等完整逻辑
- 支持多种技能类型，包括指向性技能、范围技能、持续技能等
- 实现了技能特效系统，使用粒子系统与动画系统实现技能的视觉效果

**对战规则系统**：
- 实现了小兵刷新、补兵判定、经济收益、装备购买、资源争夺等核心对战规则
- 设计了小兵定时刷新逻辑，按固定时间间隔在地图出生点生成小兵
- 实现了经济系统，对局中可获得经济资源，用于购买装备
- 实现了资源争夺功能，包括中立单位的刷新与击杀收益

**联机对战系统**：
- 实现了房间创建、玩家加入、匹配开始、实时同步等功能
- 支持断连重连机制，重连后可恢复当前对局数据
- 实现了网络质量检测功能，根据网络状况动态调整同步策略

**信息管理系统**：
- 实现了角色信息浏览功能，支持玩家对局前查看各角色属性与技能
- 实现了历史对局记录功能，记录用户的历史对局数据
- 实现了数据统计与分析功能，支持按时间筛选、查看单局详细数据

#### 2.1.4 性能优化与测试

**资源优化**：
- 采用资源池技术对高频实例化的资源进行缓存与复用，减少内存占用与GC触发频率
- 使用异步加载技术处理场景切换、技能特效加载等高耗时操作，避免阻塞主线程
- 对纹理、模型等资源进行压缩，减少资源占用与加载时间
- 实现LOD技术，对远处的模型使用低精度版本，减少渲染开销

**网络优化**：
- 采用状态同步+事件同步的混合策略，优化网络传输效率
- 对核心数据（角色位置、技能释放指令）采用高频同步（10次/秒以上）
- 对非核心数据（如小兵血量、经济数值）采用插值同步与事件触发同步结合，减少网络开销
- 实现数据序列化压缩，降低网络传输数据量

**代码优化**：
- 优化代码结构，减少冗余代码，提高代码执行效率
- 使用对象池、协程等技术优化内存使用与性能
- 采用事件驱动设计，实现模块间解耦，提高代码的可维护性

**测试验证**：
- 进行了功能测试，验证各功能模块的正确性与完整性
- 进行了性能测试，测试游戏的帧率、内存占用、网络延迟等性能指标
- 进行了网络同步测试，验证多端数据同步的一致性与稳定性
- 测试结果表明，游戏在主流PC设备上能够稳定运行，帧率保持在60FPS以上，网络同步延迟控制在100ms以内

### 3.1 系统架构设计

#### 3.1.1 架构设计原则

本项目的架构设计遵循以下原则：

- 核心业务层作为连接基础设施层与业务逻辑层的桥梁，提供标准化的核心接口

#### 3.2.2 网络同步系统实现

**系统架构设计**：

网络同步系统基于Photon PUN2框架实现，采用客户端-服务器架构，所有游戏数据都通过Photon服务器中转，确保数据的安全性与一致性。系统采用混合同步策略，结合状态同步与事件同步的优势，对核心数据采用高频状态同步，对非核心数据采用事件同步，优化网络传输效率。

网络同步系统包含以下核心模块：

1. **连接管理模块**：负责与Photon服务器的连接管理，包括连接、断开、重连等功能。采用单例模式设计，全局唯一，提供连接状态查询、事件回调等功能。

4. **事件同步模块**：负责游戏事件的同步，包括技能释放、装备购买、击杀事件等。采用事件队列机制，确保事件的有序处理。

5. **延迟处理模块**：负责网络延迟的补偿与处理，包括插值、预测、回滚等技术。通过延迟补偿机制，减少网络延迟对游戏体验的影响。

6. **冲突解决模块**：负责多端状态冲突的解决，包括时间戳、优先级等策略。通过冲突解决机制，确保多端状态的一致性。

### 3.3 技术难点与解决方案

#### 3.3.1 多端网络同步一致性

**问题描述**：
在局域网联机时，不同客户端之间的角色位置、技能状态、战斗数据同步存在延迟与误差，影响游戏体验。

**解决方案**：
- 采用基于Photon PUN的"状态同步+事件同步"混合策略
- 核心数据（角色位置、技能释放指令）采用高频同步（10次/秒以上）
- 非核心数据（如小兵血量、经济数值）采用插值同步与事件触发同步结合，减少网络开销
- 实现数据序列化压缩，降低网络传输数据量
- 设计对局数据本地缓存机制，支持断连重连后快速恢复游戏状态

**技术原理**：
- 状态同步：定期同步游戏对象的状态，确保各客户端的状态一致
- 事件同步：通过事件触发同步重要游戏事件，如技能释放、装备购买等
- 插值同步：使用插值算法平滑角色位置等连续数据的同步，减少视觉卡顿
- 预测算法：预测角色位置等数据，减少网络延迟对游戏体验的影响

#### 3.3.2 资源管理与性能优化

**问题描述**：
游戏运行过程中内存占用较高，在复杂场景或多人对战时帧率出现波动。

**解决方案**：
- 采用"资源池+异步加载"的双层优化策略
- 对高频实例化对象使用对象池技术，减少内存分配与GC开销
- 针对场景切换、技能特效加载等耗时操作，采用Unity协程实现异步加载
- 使用Unity Profiler工具实时监控内存占用与帧率变化，针对性优化资源大小
- 实现LOD技术，对远处的模型使用低精度版本，减少渲染开销

**技术原理**：
- 资源池技术：通过复用对象减少内存分配与GC开销，提高性能
- 异步加载：使用协程在后台加载资源，避免阻塞主线程，提高游戏的响应速度
- LOD技术：根据物体与相机的距离，使用不同精度的模型，减少渲染开销
- 资源压缩：对纹理、模型等资源进行压缩，减少资源占用与加载时间

#### 3.3.3 对战规则逻辑闭环

**问题描述**：
需实现"小兵刷新-补兵判定-经济收益-装备购买-属性更新-资源争夺"的完整对战逻辑链，涉及多模块数据交互与状态同步，易出现逻辑冲突或数据错乱。

**解决方案**：
- 基于"事件驱动+模块化拆分"设计
- 通过核心业务层的事件调度服务实现各模块间解耦通信
- 对战规则模块内部按"小兵机制、经济系统、资源争夺"拆分子模块，每个子模块封装独立逻辑
- 将小兵刷新间隔、经济收益数值等参数配置化，便于调试与平衡优化

**技术原理**：
- 事件驱动设计：通过事件机制实现模块间的通信，减少模块间的直接依赖
- 模块化拆分：将复杂逻辑拆分为多个子模块，每个子模块封装独立逻辑，提高代码的可维护性
- 配置化设计：将可变参数配置化，便于调试与平衡优化，减少硬编码

   - 在局域网联机时，不同客户端之间的角色位置、技能状态、战斗数据同步存在延迟与误差，影响游戏体验
   - 断连重连后，游戏状态恢复不完整，可能导致游戏数据错乱

2. **资源管理与性能优化**：
   - 游戏运行过程中内存占用较高，在复杂场景或多人对战时帧率出现波动
   - 资源加载时间较长，影响游戏的响应速度

3. **对战规则逻辑闭环**：
   - "小兵刷新-补兵判定-经济收益-装备购买-属性更新-资源争夺"的完整对战逻辑链实现复杂，易出现逻辑冲突或数据错乱
   - 游戏平衡性调整困难，需要频繁修改参数

4. **AI系统开发**：
   - AI角色的行为逻辑、路径规划、战斗策略等功能尚未实现，影响单机游玩体验
   - AI角色的智能程度较低，缺乏战术意识

5. **回放系统开发**：
   - 历史对局的回放、分析、分享等功能尚未实现，影响用户对游戏的深度体验
   - 回放数据存储与读取效率较低，可能导致回放不流畅

#### 4.1.2 非技术层面问题

1. **项目进度管理**：
   - 部分功能模块的开发进度滞后，需要调整开发计划
   - 项目进度跟踪不及时，难以准确掌握项目进展情况

2. **团队协作**：
   - 团队成员之间的沟通与协作需要进一步加强
   - 代码审查机制不完善，代码质量参差不齐

3. **测试覆盖度**：
   - 部分功能模块的测试覆盖度不足，需要加强测试力度
   - 测试自动化程度较低，测试效率不高

### 5.2 性能优化措施

#### 5.2.1 资源优化

1. **资源池技术**：
   - 对高频实例化的资源（如Buff、技能特效）使用对象池技术，减少内存分配与GC开销
   - 实现资源池的动态扩容与收缩，根据实际需求调整资源池大小

2. **异步加载**：
   - 使用Unity协程实现资源的异步加载，避免阻塞主线程
   - 实现资源的预加载与后台加载，提高游戏的响应速度

3. **资源压缩**：
   - 对纹理、模型等资源进行压缩，减少资源占用与加载时间
   - 使用纹理图集（Texture Atlas）减少Draw Call，提高渲染效率

4. **LOD技术**：
   - 实现LOD技术，对远处的模型使用低精度版本，减少渲染开销
   - 根据物体与相机的距离，动态调整模型的精度

#### 5.2.2 代码优化

1. **算法优化**：
   - 优化游戏中的算法，减少时间复杂度与空间复杂度
   - 使用更高效的数据结构，如字典、哈希表等

2. **内存管理**：
   - 减少临时对象的创建，避免频繁的内存分配与GC
   - 使用对象池技术复用对象，减少内存碎片

3. **渲染优化**：
   - 减少Draw Call，合并相同材质的物体
   - 使用批处理（Batching）技术，提高渲染效率
   - 优化 shader，减少GPU计算开销

4. **网络优化**：
   - 优化网络同步策略，减少网络传输数据量
   - 使用数据压缩技术，降低网络传输开销
   - 实现网络质量检测，根据网络状况动态调整同步策略

3. **功能模块开发**：实现了角色系统、技能系统、对战规则系统、联机对战系统、信息管理系统等功能模块，覆盖了MOBA游戏的核心玩法与基础服务需求。

4. **性能优化与测试**：采用了资源池技术、异步加载、LOD技术等性能优化手段，确保游戏在主流PC设备上能够稳定运行，帧率保持在60FPS以上，网络同步延迟控制在100ms以内。

5. **知识技能学习**：学习了Unity引擎、C#语言、Photon PUN 2框架等技术，提升了系统架构设计能力、问题解决能力、团队协作能力、项目管理能力等。

6. **职业素养培养**：培养了软件工程质量意识、职业健康与安全意识、服务意识与职业道德、社会影响评价等职业素养。

在项目开发过程中，我们遇到了多端网络同步一致性、资源管理与性能优化、对战规则逻辑闭环等技术难点，并采取了相应的解决策略。同时，我们也积累了丰富的开发经验，为未来的游戏开发项目打下了坚实基础。

### 7.2 未来展望

本项目的未来发展方向包括：

1. **功能扩展**：
   - 增加更多的角色、技能、装备、地图等内容，丰富游戏的玩法与内容
   - 实现AI系统、回放系统、成就系统等剩余功能模块
   - 增加社交功能，如好友系统、排行榜系统等

2. **技术升级**：
   - 引入更多先进的技术，如AI技术、物理引擎、图形渲染技术等，提升游戏的品质与体验
   - 优化网络同步机制，减少网络延迟，提高多端数据同步的一致性
   - 进一步优化资源管理与性能，减少内存占用与帧率波动

3. **平台扩展**：
   - 将游戏扩展到更多平台，如移动平台、主机平台等，扩大游戏的用户群体
   - 实现跨平台联机功能，支持不同平台的玩家一起游戏

4. **社区建设**：
   - 建立游戏社区，鼓励玩家参与游戏的开发与改进，形成良好的游戏生态
   - 提供 mod 支持，允许玩家自定义游戏内容
   - 组织线上线下比赛，增强玩家的参与感与归属感

5. **商业化探索**：
   - 探索游戏的商业化模式，如道具收费、广告收入等，实现游戏的可持续发展
   - 开发周边产品，如手办、漫画等，扩大游戏的影响力
   - 与其他IP合作，开发联动内容，吸引更多玩家
